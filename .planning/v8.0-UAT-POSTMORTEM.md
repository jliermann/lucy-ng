## Bottom line

The agent reached the **correct structure** (Ibuprofen, RDKit-verified, MAE 0.256) — **but by bypassing the entire v8.0 pyLSD system**, not by using it. The "green checkmark" is misleading: v8.0's deliverable (pyLSD multi-run for 4J handling) was buggy and abandoned mid-run; the solution came from the pre-v8.0 manual approach (manual 4J removal + forced benzene fragment + ~7 coordinator interventions + direct lsd binary).

**UAT verdict: FAIL as a validation of v8.0** (correct chemistry achieved via the wrong mechanism).

## Inconsistencies by severity

### CRITICAL — v8.0 purpose not achieved
1. **pyLSD merge bug (Phase 67):** SolutionMerger ingests 0 despite thousands of per-permutation raw solutions. Root cause = outlsd header-conversion bug (Phase 65 `_run_outlsd`).
2. **Solution found via bypass:** final answer via direct lsd binary, K=0, no permutations, no merged.smi. The v8.0 system contributed nothing to the solution.
3. **Phase 65 hypothesis contradicted:** "remove 4J → aromatic ring appears" did NOT hold. At iter2 (4J handled) **0/90 solutions were aromatic**. The benzene ring only appeared in iter3 when explicitly **forced via a `SKEL` benzene fragment**. (Partially confounded: iter2's 90 were computed with SYME/DEFF NOT dropped.)
4. **Constraint-loss bug (v3.0 pattern) recurred — now in pyLSD tooling:** permutation files drop BOND/SYME/DEFF NOT/grouped (`perm_00/compound.lsd` = 542 B, HMBC-only).

### MAJOR — skill-vs-tool contradiction
5. **SYME and DEFF NOT are NOT native LSD-3.4.9 commands** (errors 102/150). Skills teach them; the binary rejects them. lucy-ng translates internally, but the bug-forced direct-binary path loses them. (The deleted CASE1-namespace memory had documented this correctly — it was wrongly dismissed as "stale".)
6. **`lucy lsd run` / `lucy pylsd run` broken** (exit 255, stdin mode → no `.sol`). Forced manual outlsd workarounds throughout.

### PROCESS / RELIABILITY
7. **Solution driven by ~7 manual coordinator interventions**, not the autonomous pipeline (multiplicity correction, acid-vs-ester, H-mapping, pyLSD bypass, forced benzene, ethyl-isomer catch via RDKit, ortho exclusion).
8. **Iter1 file modified after devils-advocate validation** (protocol violation — exactly what DA exists to prevent).
9. **nmr-chemist initial multiplicity error** (44.90 as CH3 vs CH) — caught by coordinator.
10. **Inventory documentation mismatches** (deff_not_patterns vs actual DEFF F1/F2; deferred_4j ≠ [] when pylsd_mode=false).

## Root causes (consolidated)

- **R1 — outlsd conversion / `lucy lsd run` plumbing** (Phase 65/69): solutions exist (solncounter) but never become SMILES. This is foundational — everything downstream (merge, ranking) depends on it. Likely THE keystone bug.
- **R2 — permutation generation drops non-HMBC constraints** (Phase 67): BOND/SYME/DEFF NOT/grouped lost in perm files.
- **R3 — SYME/DEFF NOT are lucy-ng abstractions, not native LSD** — the constraint model assumes a translation layer that the direct-binary / permutation path bypasses.
- **R4 — skill/architecture:** normal-LSD vs pyLSD documentation imbalance may cause the agent to revert to the better-documented (normal LSD) path (user hypothesis — testable). The pipeline is not autonomous enough; the result depended on coordinator rescue.

---

## Open design questions (MUST be answered before any fix sequencing)

1. **Is pyLSD multi-run the right 4J approach at all?** Phase 65's premise was contradicted (ring had to be forced). Decide: keep pyLSD-permutation 4J handling, or pivot the 4J strategy.
2. **Single path vs dual path?** Should normal-LSD and pyLSD coexist, or should there be ONE clear solver path to stop the agent reverting? (Tests user's doc-imbalance hypothesis.)
3. **Where does constraint translation live?** If SYME/DEFF NOT are non-native, the native-translation must flow losslessly into permutation generation AND the direct-binary path — architectural decision.
4. **How is the aromatic ring established?** Forced fragment (SKEL) vs emergent from constraints — and if forced, how does the agent decide the ring system de novo?
