# v7.0 Statistical 4J Detection — Roadmap

**Goal:** Build database-backed statistical detection of 4J CH HMBC couplings so the CASE pipeline can identify and exclude them before LSD solving, enabling correct structure determination for aromatic compounds.

**Phases:** 59-64 (6 phases)
**Requirements:** DB-01..04, GEN-01..04, DET-01..05, CLI-01..03, AGT-01..04, VAL-01..04

## Phase 60: Statistics Generator

**Goal:** CouplingPathStatsGenerator that populates coupling_path_stats from 928K compounds.

**Requirements:** GEN-01, GEN-02, GEN-03, GEN-04, VAL-04

**Plans:** 2 plans

Plans:
- [ ] 60-01-PLAN.md — CouplingPathStatsGenerator core algorithm with checkpoint/resume
- [ ] 60-02-PLAN.md — CLI command wiring and integration tests

**Scope:**
- `CouplingPathStatsGenerator` class in prediction/coupling_path_generator.py
- Algorithm: parse SMILES → GetDistanceMatrix → enumerate CH pairs → HOSE R2 for both → accumulate
- Checkpoint/resume via operation_checkpoint table (every 10K compounds)
- COCONUT 1-based → 0-based atom index conversion
- Skip compounds with NULL atom_index, validate C symbol
- CLI: `lucy database generate-coupling-stats [--db PATH] [--resume/--fresh] [--limit N]`
- `--limit N` for development (generate from first N compounds only)
- No explicit hydrogens (project convention)

**Success criteria:**
- Generation on 1K compounds completes in < 30 seconds
- Checkpoint recovery works (kill + restart produces same result)
- Generated rows have reasonable distribution (2J/3J/4J counts)
- `lucy database info` shows populated coupling_path_stats

**Note:** Full 928K generation (estimated 13-26 hours) runs after code ships. Development uses `--limit 10000`.

## Phase 61: Detection Engine

**Goal:** StatisticalDetector methods and CLI commands for 4J risk scoring.

**Requirements:** DET-01, DET-02, DET-03, DET-04, DET-05, CLI-01, CLI-02, CLI-03

**Scope:**
- `CouplingPathDistribution` model: j2, j3, j4, j5_plus probabilities
- `CouplingPathResult` model: carbon_shift, h_carbon_shift, distribution, total_observations, risk_level, recommendation
- `StatisticalDetector.detect_4j_coupling()`: shift window → HOSE codes → coupling_path_stats query → aggregate → classify
- `StatisticalDetector.detect_4j_batch()`: optimized batch scoring
- Three-tier classification with configurable thresholds
- Minimum observation threshold (50) → "insufficient data"
- CLI `lucy detect 4j` and `lucy detect 4j-batch` with --format json
- Fallback: if exact HOSE pair has no data, aggregate over all partners for that carbon HOSE

**Success criteria:**
- `lucy detect 4j 129.38 45.03 --format json` returns valid risk assessment (after stats populated)
- `lucy detect 4j-batch --correlations "129.38:45.03,127.26:44.90" --format json` returns batch results
- Three-tier classification works with test data
- Low observation count returns "insufficient data"

**Depends on:** Phase 59 (schema), Phase 60 (populated data for integration tests)
