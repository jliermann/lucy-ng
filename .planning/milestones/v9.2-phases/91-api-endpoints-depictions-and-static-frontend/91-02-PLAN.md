<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → API handler | Untrusted-content files (timing.jsonl, CASE-PROGRESS.md) written by the CASE run are read and parsed; a malformed/partial file must not crash the endpoint |
| localhost client → API | Read-only GET requests on 127.0.0.1 (bind established in Phase 90); no mutating input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-91-02 | Denial of Service | GET /api/log reading a very large CASE-PROGRESS.md | accept | Localhost-only, single-user dev monitor; log files are bounded by run length; no amplification. Documented as LOW per RESEARCH §Security Domain |
| T-91-03 | Tampering | Mid-write/partial timing.jsonl parsed by status | mitigate | Line-by-line json.loads with per-line try/except skip (Pitfall 7); all reads wrapped so no 500 escapes |
| T-91-04 | Information Disclosure | Exception detail leaking file paths in a 500 | mitigate | No endpoint raises: every file read degrades to a 200 waiting payload (D-04) |
| T-91-SC | Tampering | package installs | accept | No new packages this phase (stdlib + already-present fastapi) |
</threat_model>
