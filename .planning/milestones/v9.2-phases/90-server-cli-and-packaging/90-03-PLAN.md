<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shell user → `lucy webview <cmd> <analysis_dir>` | User-supplied path + flags enter the CLI |
| serve command → server.start subprocess | CLI hands args to the lifecycle engine which launches a detached process |
| core install → webview commands | Extra may be absent; import must not crash the core CLI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-90-08 | Tampering | Path traversal via `analysis_dir` CLI arg | mitigate | Dispatched to server.start which does `Path.resolve()` + `is_dir()` check (plan 90-02 T-90-03) |
| T-90-09 | Tampering | Subprocess arg injection through CLI flags | mitigate | server.start uses list-form Popen, no shell=True (plan 90-02 T-90-04); `_run` args are typed Click options |
| T-90-10 | Elevation of Privilege | `_run` invoked directly by a confused user | accept | Hidden subcommand; binds 127.0.0.1 only; read-only skeleton with docs disabled |
| T-90-11 | Denial of Service | Core install crashes on `lucy --help` due to accidental fastapi import | mitigate | `_require_webview()` lazy guard + import-safety test asserts `fastapi not in sys.modules` after importing the core CLI |
| T-90-SC | Tampering | pip installs (fastapi, uvicorn) via the webview extra | mitigate | Both audited [OK] in 90-RESEARCH Package Legitimacy Audit (slopcheck 0.6.1, 2026-07-02); no [ASSUMED]/[SUS]/[SLOP] — no blocking human checkpoint required |
