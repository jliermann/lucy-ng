# Milestone v1.2: HOSE Database Prediction

**Status:** ✅ SHIPPED 2026-01-18
**Phases:** 16-19
**Total Plans:** 4

## Overview

Replace in-memory HOSE lookup with database-backed prediction using extended COCONUT dataset (~895K compounds) for improved accuracy and confidence scoring. The compound database now powers both dereplication AND 13C prediction with 7.9M pre-computed HOSE statistics.

## Phases

### Phase 16: Database Schema ✅
**Goal**: Add hose_stats table to compounds.db for HOSE-based shift prediction
**Depends on**: Milestone v1.1 complete
**Completed**: 2026-01-15

Plans:
- [x] 16-01: Database schema for HOSE stats (3 tasks)

**Details:**
- hose_stats table with precomputed statistics (mean, std, count) per HOSE code at each radius
- Composite primary key (hose_code, radius) for uniqueness
- Index on hose_code for O(1) lookups
- HOSEStatsRecord Pydantic model
- DatabaseManager methods for CRUD operations
- Schema version bumped to 2

## Milestone Summary

**Key Decisions:**
- Protocol pattern for interchangeable backends (HOSELookupProtocol)
- Database-first: Auto-detection prefers database over JSON table
- Store pre-computed stats (mean/std/count) not raw observations
- Welford's algorithm for incremental statistics in resumable generator

**Key Accomplishments:**
1. 7.9M HOSE statistics from 895K compounds (radii 1-6)
2. DatabaseHOSELookup adapter for O(1) database queries
3. C13Predictor with dual-backend support
4. CLI --db option with intelligent auto-detection
5. MCP get_hose_stats_info tool for agent capability checking
6. Resumable HOSE generation with checkpoint/resume

**Issues Resolved:**
- Memory efficiency for large dataset processing (Welford's algorithm)
- Correct atom indexing from COCONUT SDF (1-based vs 0-based)
- Single database approach simplifies AI agent instructions

**Technical Debt Incurred:**
- None significant
