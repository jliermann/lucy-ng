### Agent Behavior Gaps Found

**Gap A: Badlist dropped across iterations**
- Agent wrote 6 DEFF NOT patterns in iteration 1 LSD file
- When rebuilding LSD for iterations 2-4, DEFF NOT was not carried forward
- Evidence: iteration_01/compound.sol preserves original input with DEFF NOT
- Impact: 3 strained-ring solutions (cyclopropanone, cyclobutenone) survived in final 13

**Gap B: Signal grouping detected but never applied**
- Agent correctly ran `lucy analyze grouping` and found [44.90, 45.03] group
- Used only for confidence annotation in final_results.md
- Never translated to SYME or parenthesized HMBC in any LSD file

**Gap C: Grouped atom notation lost**
- Iteration 1 correctly used `HMBC 2 (6 7)` for equivalent atoms
- Iterations 2-4 reverted to simple `HMBC 2 6`
- Agent overwrote iteration_01/compound.lsd after running, removing grouped notation

**Root cause pattern:** Agent builds constraints correctly for iteration 1, then reconstructs LSD from scratch each iteration, losing auxiliary constraints (DEFF NOT, grouped notation, signal grouping). The iterative HMBC-addition workflow needs to preserve non-HMBC constraints across iterations.

## Conclusion

**All 17 requirements are satisfied at the code/tool level.** The detection CLI commands work correctly, the database is fully populated, two-tier ranking prevents MAE hallucination, and the agent file contains comprehensive detection protocol knowledge.

**The gaps are in agent behavior, not tools.** The CASE agent has the knowledge to use all v3.0 features but doesn't consistently apply them across LSD iterations. This is an agent prompting/workflow issue, not a code deficiency.

**Recommendation:** Complete v3.0 milestone, document agent behavior gaps as tech debt for next milestone (v3.1 or v4.0). The tools are shipped and validated; the agent workflow refinement is a separate concern.
