# Phase 2: 1D NMR Reading — Plan 01

## Goal

Implement a Bruker 1D file reader that loads processed NMR data and returns populated `Spectrum1D` objects.

## Context

- nmrglue library selected in Phase 1 with documented API
- `Spectrum1D` model exists with fields: data, ppm_scale, nucleus, frequency, solvent, metadata
- Test data available: Ibuprofen dataset with 1H (exp 1) and 13C (exp 2) spectra
- Key nmrglue API: `bruker.read()` returns (dic, data), `bruker.make_uc()` for ppm scale

## Tasks

### Task 1: Create BrukerReader class with read_1d method

Create `src/lucy_ng/readers/bruker.py` with:

```python
class BrukerReader:
    """Reader for Bruker NMR data files."""

    @staticmethod
    def read_1d(experiment_dir: str | Path) -> Spectrum1D:
        """Read a Bruker 1D experiment directory and return Spectrum1D."""
```

Implementation:
- Use `nmrglue.bruker.read_pdata()` to read processed data from pdata/1/
- Extract nucleus from `dic['acqus']['NUC1']` (strip angle brackets)
- Extract frequency from `dic['acqus']['SFO1']`
- Extract solvent from `dic['acqus']['SOLVENT']` (strip angle brackets)
- Extract pulse program from `dic['acqus']['PULPROG']` for metadata
- Generate ppm scale using `nmrglue.bruker.make_uc(dic, data).ppm_scale()`
- Return populated `Spectrum1D` instance

**Acceptance**: Function reads Bruker directory and returns Spectrum1D.

### Task 2: Handle parameter extraction edge cases

Enhance parameter extraction to handle:
- Missing optional parameters (solvent may be absent)
- Angle bracket stripping for string parameters (`<CDCl3>` -> `CDCl3`)
- Validation that NUC1 is a supported nucleus (1H, 13C, etc.)
- Store additional metadata: pulse program, number of scans (NS), temperature (TE)

**Acceptance**: Reader handles missing/malformed parameters gracefully.

### Task 3: Write tests with real Ibuprofen data

Create `tests/test_bruker_reader.py` with:

1. `test_read_1h_spectrum`: Load Ibuprofen exp 1, verify:
   - nucleus == "1H"
   - frequency ~= 499.87 MHz
   - solvent == "CDCl3"
   - data array is not empty
   - ppm_scale has same length as data

2. `test_read_13c_spectrum`: Load Ibuprofen exp 2, verify:
   - nucleus == "13C"
   - frequency ~= 125.71 MHz
   - data and ppm_scale populated

3. `test_invalid_directory`: Verify appropriate error for non-existent path

4. `test_ppm_scale_range`: Verify 1H ppm range is reasonable (roughly 0-14 ppm)

**Acceptance**: All tests pass with real NMR data.

### Task 4: Export BrukerReader from readers module

Update `src/lucy_ng/readers/__init__.py` to export `BrukerReader`.

Update `src/lucy_ng/__init__.py` to include reader in package exports.

**Acceptance**: `from lucy_ng.readers import BrukerReader` works.

### Task 5: Run full test suite

Run `pytest` to verify:
- All new tests pass
- All existing tests still pass (no regressions)
- Reader integrates with existing Spectrum1D model

**Acceptance**: `pytest` exits with status 0, all tests green.

## Risks

- **nmrglue API differences**: The `read_pdata` vs `read` distinction may have nuances. Mitigation: test with real data first.
- **File structure variations**: Different Bruker versions may have slightly different structures. Mitigation: start with known good data, expand later.
