<objective>
Wire the ranker CLI to the shared DB-first resolver so `lucy lsd rank` uses the SAME backend
as `lucy predict c13`, add `--db`/`--max-radius` parity to the `rank` command, route
`predict c13` through the same helper, and pin the fix with RANK-01/02/03 regression tests on
CASE1 (ibuprofen) and CASE3 (pulegone).

Purpose: Close the defect — `lucy lsd rank` reported 2.23 MAE / 8-of-13 for ibuprofen while
`predict c13` reported 0.24 / 13-of-13 on the SAME molecule, because the ranker was hardwired
to the sparse NMRShiftDB-only JSON table. Pointing it at the DB via the shared helper makes the
two paths identical (RANK-01), brings the MAE into agreement (RANK-02), and the regression test
locks it (RANK-03).

Output: edited `cli/lsd.py` (resolver-wired `_perform_ranking`, new `--db`/`--max-radius`
options on `lsd_rank`), `cli/predict.py` routed through the same helper, and three regression
tests in `tests/test_ranking.py`.
</objective>
