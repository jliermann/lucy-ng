    - LSDCorrelation(1, 2, "HMBC", min_bonds=2, max_bonds=3).to_lsd_line() == "HMBC 1 2" (unchanged default)
    - LSDCorrelation(5, 8, "HMBC", min_bonds=3, max_bonds=4).to_lsd_line() == "HMBC 5 8 3 4"
    - LSDCorrelation(1, 1, "HSQC").to_lsd_line() == "HSQC 1 1" (unaffected)
    - LSDCorrelation(2, 5, "COSY").to_lsd_line() == "COSY 2 5" (unaffected)
  </behavior>
  <implementation>
    In LSDCorrelation.to_lsd_line(), change the HMBC branch:

    ```python
    elif self.correlation_type == "HMBC":
        if self.min_bonds != 2 or self.max_bonds != 3:
            return f"HMBC {self.atom1_index} {self.atom2_index} {self.min_bonds} {self.max_bonds}"
        return f"HMBC {self.atom1_index} {self.atom2_index}"
    ```

    CRITICAL: Use OR condition (not AND). AND would never trigger because defaults are min=2, max=3.

    Also add two new fields to LSDProblem dataclass (after `comments`):

    ```python
    pylsd_mode: bool = False
    elim_commands: list[tuple[int, int]] = field(default_factory=list)
    ```

    These fields are consumed by generator.py (Plan 02). pylsd_mode gates FORM/ELIM emission. elim_commands stores (N, M) pairs for ELIM lines.
  </implementation>
