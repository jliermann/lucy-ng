```yaml
---
name: lucy-ng:supervisor
description: >
  Orchestrates CASE workflow by selecting the right specialist skill.
  Routes between dereplication, full CASE, sanitize, and blind-case based on
  user intent. Detects unproductive loops and escalates to user.
---
```

**Content sections:**

1. **Workflow Selection** (~15 lines, from CLAUDE.md section 3.1):
   Decision tree for which skill to invoke:
   - Is this for blind CASE evaluation on public data? -> `/lucy-ng:sanitize` (then fresh session)
   - Do you want to check databases first? -> `/lucy-ng:dereplicate`
     - Match found? -> Done
     - No match? -> `/lucy-ng:CASE`
   - Skip to full CASE? -> `/lucy-ng:CASE`
   - Default: dereplication first, then CASE if no match

2. **Loop Detection Patterns** (~15 lines, placeholder for Phase 24):
   Table of loop patterns and interventions:
   | Loop Type | Detection | Intervention |
   | 0-solution loop | 2+ LSD runs return 0 solutions with same approach | Invoke diagnostic: check sp2 count, H budget, correlation order |
   | Solution explosion | 2+ runs return >100 solutions | Check ELIM usage, add more HMBC constraints |
   | ELIM thrashing | ELIM added/removed 2+ times without diagnosing root cause | Stop and diagnose: sp2 count, H budget first |

3. **Escalation Criteria** (~10 lines, from CLAUDE.md section 14.5 "When to Ask for Help"):
   - Conflicting data between experiments
   - Unusual chemical shifts outside normal ranges
   - Molecular formula doesn't match observed data
   - 3 failed intervention cycles with same pattern -> escalate to user
   - User requests interpretation beyond available data

**Note:** This is a FOUNDATION document. Phase 24 will significantly expand it with full supervisor agent implementation. Keep it focused and minimal now.
