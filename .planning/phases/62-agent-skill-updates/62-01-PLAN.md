Run ONCE during setup, on ALL HMBC correlations (after guided picking, before sending [SETUP-COMPLETE]). This is a single batch call, not per-correlation.

- Do NOT remove likely_4j correlations from the HMBC list silently. Report them with risk scores so lsd-engineer can make deferral decisions.
- insufficient_data is NOT the same as unlikely — flag it separately so the team knows confidence is low.

### 4J Deferral Rule (Statistical)

When nmr-chemist provides 4J risk scores in [SETUP-COMPLETE], handle each tier:

**likely_4j (P >= 0.50):** DEFER. Never include in early HMBC batches. Add to `deferred_4j` inventory with risk_level and probability. Only add as final batch if solutions > 10 after all other correlations exhausted.

**possible_4j (0.15 <= P < 0.50):** Use extended bond range. Write as `HMBC X Y 2 4` (allows 2-4 bond paths instead of default 2-3). Include in normal batch ordering — do NOT defer these.

**insufficient_data:** Treat as normal HMBC (default 2-3 bond). Note uncertainty in inventory.

**Deferral cap:** Maximum 3-4 correlations may be deferred as likely_4j. If more than 4 correlations are likely_4j, defer only the top 4 by probability (highest P first). Remaining likely_4j correlations use `HMBC X Y 2 4` instead (treated as possible_4j). This prevents combinatorial explosion from too many deferred constraints.
```
