**Content structure (use `<objective>` and `<process>` with `<step>` XML tags like dereplicate.md):**

1. **Objective** -- AI-driven sanitisation of Bruker NMR datasets. Explicitly state (SANT-01): "There is NO CLI command for sanitisation. This is a pure AI task requiring semantic reasoning to identify compound identifiers that reveal chemical identity."

2. **Step: validate_input** -- Parse user argument (dataset path). Verify path exists and contains Bruker NMR data (look for numbered experiment directories containing acqus files). If no argument given, show usage: `/lucy-ng:sanitise <dataset_path>`.

3. **Step: discover_text_files** -- Use Glob with `**/*` pattern on dataset path. Classify files:
   - SKIP (binary NMR data): fid, ser, 1r, 1i, 2rr, 2ri, 2ir, 2ii, 3rrr and all 3D variants
   - SKIP (binary extensions): .pdf, .png, .jpg, .jpeg, .gif, .tiff, .bmp
   - DELETE (structure files): *.mol, *.sdf, *.cdx, *.cml
   - DELETE (audit logs): audita.txt, auditp.txt
   - READ (text files): title, acqus, acqu2s, procs, proc2s, *.xml, *.txt, pulseprogram, and any other non-binary files

4. **Step: delete_structure_and_audit_files** -- Use Bash `rm` to delete all structure files (*.mol, *.sdf, *.cdx, *.cml) and audit logs (audita.txt, auditp.txt). These files contain compound structures and user activity that must be removed entirely, not redacted. Report what was deleted.

5. **Step: scan_and_redact** -- For each text file from the discovery step:
   - Read the file content
   - Apply semantic reasoning to identify compound identifiers (see identifier categories below)
   - If identifiers found: replace each with `[REDACTED]` and Write the file back
   - Track all changes (file path, original text, replacement) for the report

   **Identifier categories to detect (SANT-02):**

   a. **Chemical names** (semantic understanding required):
      - IUPAC systematic names: "2-(4-isobutylphenyl)propionic acid"
      - Trivial/common names: "Ibuprofen", "Caffeine", "Indigo", "Aspirin"
      - Trade names: "Advil", "Brufen", "Tylenol"
      - Include all case variants: "Ibuprofen", "ibuprofen", "IBUPROFEN"

   b. **Database identifiers:**
      - CAS numbers: format xxxxxxx-xx-x (e.g., "482-89-3", "CAS 58-08-2")
      - InChI strings: starts with "InChI=1S/"
      - InChIKey: 27-character format XXXXXXXXXXXXXX-YYYYYYYYYYY-Z
      - PubChem/ChEBI IDs: "CID 12345", "CHEBI:12345"

   c. **SMILES strings** (best-effort, LOW priority):
      - AI may recognise SMILES notation in text but this is not high priority
      - If missed, acceptable -- exploiting SMILES for CASE cheating requires significant effort

   d. **File paths containing compound names:**
      - Example: `Z:/Torsten/NMR/data/spek-vl_premium/nmr/Ibuprofen/1/pdata/1/procs`
      - Redact only the compound name portion within the path

   e. **Dataset naming patterns:**
      - nmrXiv patterns: "Classics_CompoundName", "nmrXiv_Caffeine_01"
      - Lab patterns: "JD_caffeine_01"

   **Do NOT redact (false positive avoidance):**
   - Experiment types: 1H, 13C, HSQC, HMBC, COSY, DEPT, NOESY
   - Solvents: CDCl3, DMSO-d6, D2O, MeOD
   - Software names: TopSpin, nmrPipe, MestReNova
   - Instrument parameters: frequency values, sweep widths, dates
   - Generic placeholders: "Unknown", "Sample", "Test"

6. **Step: present_report** -- After redaction, present a structured report:
   ```
   ## Sanitisation Report

   Dataset: <path>

   ### Files Deleted
   - [list of deleted structure/audit files]

   ### Identifiers Redacted
   | File | Identifier | Type |
   |------|-----------|------|
   | 1/pdata/1/title | Ibuprofen | Chemical name (trivial) |
   | 1/acqus | Ibuprofen | File path component |
   ...

   ### Directory Name Warning
   (If the dataset directory itself contains a compound name, warn the user: "Dataset directory name 'X' may contain a compound identifier. Rename manually after sanitisation if needed.")
   ```

7. **Step: verify_completeness** (SANT-04) -- Full re-scan verification. Re-read ALL text files in the dataset (same files as discovery step). Apply the same semantic identifier detection. Report:
   - If 0 identifiers found: "Verification PASSED -- no compound identifiers detected in second pass"
   - If identifiers found: "WARNING: Found N identifiers missed in first pass" and list them, then redact them and re-verify

8. **Step: handoff_instructions** -- After verification passes, instruct user:
   - "Start a NEW AI session for CASE analysis (this session has seen the compound identity)"
   - "Provide the molecular formula to the CASE AI (simulating HRMS data)"
   - "Do not mention compound identity to the CASE AI"
