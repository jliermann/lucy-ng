<objective>
Create CouplingPathStatsGenerator that iterates compounds from the database, computes shortest bond-path distances for all (carbon, proton-bearing-carbon) pairs via RDKit GetDistanceMatrix, generates HOSE codes at radius 2 for both atoms, and accumulates (carbon_hose, h_carbon_hose, bond_distance) -> count statistics. Includes checkpoint/resume via operation_checkpoint table every 10K compounds.

Purpose: This is the data pipeline that populates the coupling_path_stats table (created in Phase 59) with the statistical foundation for 4J HMBC coupling detection.
Output: `CouplingPathStatsGenerator` class with `generate_all()` and `populate_database()` methods, plus comprehensive tests.
</objective>

    Create `src/lucy_ng/prediction/coupling_path_generator.py` with `CouplingPathStatsGenerator` class:

    1. Constructor: takes `DatabaseManager`, creates `HOSECodeGenerator` instance.

    2. `generate_all(progress=True, limit=None)` method:
       - Iterate compounds via `db_manager.iter_compounds_with_shifts()` (or `_from()` for resume)
       - If `limit` is set, stop after N compounds (for development/testing)
       - For each compound: parse SMILES via `HOSECodeGenerator.prepare_mol(smiles)` (implicit H, NO AddHs!)
       - Build atom_index -> shift_ppm mapping from shifts list
       - Skip compound if ANY shift has NULL atom_index (architecture doc says skip these)
       - COCONUT 1-based conversion: `atom_idx_0based = atom_index - 1` (project convention from CLAUDE.md)
       - Validate each atom: `mol.GetAtomWithIdx(atom_idx).GetSymbol() == 'C'` — skip non-carbons
       - Use `Chem.GetDistanceMatrix(mol)` to get all-pairs shortest paths in one call
       - Identify proton-bearing carbons: `atom.GetTotalNumHs() > 0` (uses implicit H count)
       - For each pair (carbon_c, carbon_h) where carbon_c != carbon_h and carbon_h is proton-bearing:
         - Get distance from distance matrix: `dist_matrix[c_idx][h_idx]`
         - Cap at 5 for 5+ bond paths. Skip if distance is 0 or infinity (disconnected)
         - Generate HOSE R2 for both: `hose_gen.generate_for_atom(mol, c_idx, radius=2)` and same for h_idx
         - Skip pair if either HOSE code generation fails
         - Accumulate: `counts[(c_hose, h_hose, int(distance))] += 1`
       - Return the counts dict
       - Use tqdm progress bar if `progress=True`

    3. `populate_database(progress=True, limit=None)` method:
       - Call `generate_all()` to get counts dict
       - Convert to list of `CouplingPathStatsRecord` objects
       - Call `db_manager.insert_coupling_path_stats_batch(records)`
       - Return inserted count

    4. Properties: `compounds_processed`, `compounds_failed` (same pattern as BondPairStatsGenerator)

    Follow the BondPairStatsGenerator pattern closely (simpler than ResumableHOSEStatsGenerator — no Welford needed, just counting).
