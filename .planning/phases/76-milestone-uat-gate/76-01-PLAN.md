<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| argv→script | formula string supplied as CLI argument — attacker-controlled string parsed as a formula |
| file→script | merged.smi path and content — attacker-controlled file whose lines are parsed as SMILES |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-76-01 | Tampering | merged.smi SMILES lines | mitigate | `Chem.MolFromSmiles()` returns None on malformed SMILES; None entries are skipped, never passed to CalcMolFormula or GetIsAromatic. Script never crashes on invalid SMILES. |
| T-76-02 | Spoofing | formula argv | mitigate | Formula used only as a plain string equality comparison (`==`). No parsing, no eval, no regex exec on user-supplied formula. Worst case: wrong pass/fail result, no code execution. |
| T-76-03 | Denial of Service | Very large merged.smi (millions of lines) | accept | Script is a local developer tool; production DoS is out of scope. Top-3 slice via parse logic means only first ~3 valid lines are checked — effectively bounds work. |
| T-76-04 | Information Disclosure | Script exit code leaks pass/fail to shell | accept | Intentional by design (D-76-03); exit code is the primary machine-readable interface. |
| T-76-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan. RDKit is already in requirements; no new dependencies introduced. |
</threat_model>
