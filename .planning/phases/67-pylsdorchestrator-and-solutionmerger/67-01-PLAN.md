    RED phase — write these tests first (all must fail):

    1. test_generates_permutation_files: Create an LSDProblem with 5 HMBC correlations, 3 flagged as suspects. Call orchestrator.run() (mock LSDRunner.run_file and outlsd). Assert exactly 8 permutation directories created (perm_00 through perm_07), each containing an LSD file.

    2. test_permutation_content: For a problem with 2 suspects (atom pairs 4-8 and 6-9), verify:
       - Permutation (True, True): both suspects present with max_bonds=4
       - Permutation (True, False): first suspect present (max_bonds=4), second absent
       - Permutation (False, False): neither suspect present
       All non-suspect correlations remain unchanged in every permutation.

    3. test_k_greater_than_3_raises: Call orchestrator.run() with 4 suspect correlations. Assert ValueError raised with message containing "Too many suspect correlations". Assert no directories created.

    4. test_k_boundary: K=3 succeeds (8 permutations). K=0 succeeds (1 permutation, base problem unchanged). K=1 produces 2 permutations.

    5. test_outlsd_bypass: Verify orchestrator calls subprocess.run([outlsd_path, "5"], stdin=...) for each permutation with a .sol file. Do NOT call LSDRunner._run_outlsd().

    6. test_permutation_result_structure: Each PermutationResult has perm_index (int), include_flags (list[bool]), suspect_correlations, lsd_result, smiles_file (Path|None), perm_dir (Path).

    GREEN phase — implement:
    - Dataclasses: PermutationResult, OrchestrationResult in orchestrator.py
    - PyLSDOrchestrator.__init__(lsd_path, timeout=120): creates LSDRunner
    - PyLSDOrchestrator.run(base_problem, suspect_correlations, output_dir): K>3 check FIRST, then itertools.product permutations, deepcopy + filter/re-add suspects, write_file, run_file, _run_outlsd
    - PyLSDOrchestrator._build_permutation(base_problem, suspects, include_flags): deepcopy, remove suspects by (atom1_index, atom2_index, correlation_type) tuple, add back included with max_bonds=4
    - PyLSDOrchestrator._run_outlsd(perm_dir, lsd_file): direct subprocess.run([outlsd_path, "5"], stdin=sol_file) — bypasses buggy LSDRunner._run_outlsd

    CRITICAL anti-patterns from research:
    - Do NOT identify suspects by id() after deepcopy — use (atom1_index, atom2_index, correlation_type) tuple
    - Do NOT use LSDRunner._run_outlsd() — it has a known bug (missing mode argument)
    - Do NOT mutate base_problem directly — always deepcopy first
    - Check K>3 BEFORE any directory creation
  </behavior>
