  <behavior>
    **expand_query_fingerprint(fp: bytes, expand_bins: int = 1) -> bytes:**
    - Input: 32-byte fingerprint from shifts_to_fingerprint, expand_bins count
    - Output: 32-byte fingerprint with each set bit expanded to ±expand_bins neighbors
    - Edge cases: bits at index 0 and 255 don't wrap around
    - Example: single bit at index 10 with expand_bins=1 -> bits 9, 10, 11 all set
    - Unchanged bits: zero bytes stay zero

    **FragmentSearcher(db_path) — context manager wrapping FragmentDatabaseManager:**
    - __init__ stores db_path, does NOT open connection or load data
    - __enter__ opens FragmentDatabaseManager connection
    - __exit__ closes connection

    **FragmentSearcher.search(experimental_shifts, dev_threshold=2.0, avgdev_threshold=1.0, max_results=20, min_atom_count=3, verbose=False) -> list[SSCMatch]:**
    1. Build query fingerprint via shifts_to_fingerprint(experimental_shifts)
    2. Expand it via expand_query_fingerprint(query_fp)
    3. Pre-screening: iterate batches from iter_ssc_bitsets(batch_size=100_000). For each batch, use NumPy vectorised AND: convert batch to (N,32) ndarray, compute mask = np.all((batch_fps & q_arr) == batch_fps, axis=1). Collect passing ssc_ids.
    4. Fine matching: fetch SSCRecords via get_ssc_by_id (chunk into batches of 999 for SQLite placeholder limit). For each record where atom_count >= min_atom_count, run greedy nearest-neighbour matching:
       - Sort fragment shifts ascending
       - For each fragment shift, find closest remaining query shift
       - If per-pair abs(query - fragment) > dev_threshold, reject
       - After all pairs matched, compute avg_deviation = mean of absolute deviations
       - If avg_deviation > avgdev_threshold, reject
       - Otherwise create SSCMatch with ssc_id, smiles, atom_count, avg_deviation, matched_shifts, fragment_shifts
    5. Ranking: sort matches by (-atom_count, avg_deviation). Assign rank 1..N. Return top max_results.
    6. If verbose=True, print pre-screen count and fine-match count to stderr.

    **Key constraints:**
    - Do NOT load all bitsets into RAM at startup (use iter_ssc_bitsets streaming)
    - Multiplicity matching is NOT implemented (SSCRecord has no per-shift multiplicity data) — skip silently
    - get_ssc_by_id called with chunks of up to 999 IDs (SQLite parameter limit is 999)
  </behavior>
