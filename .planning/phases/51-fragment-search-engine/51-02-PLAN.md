  <action>
Add a `search` command to the existing `fragment` Click group in `cli/fragment.py`. Follow the existing `lucy lsd rank` and `lucy detect hybridisation` CLI patterns.

**Click options:**
- `--shifts` (required, str): Comma-separated 13C chemical shifts in ppm
- `--db` / `db_path` (Path, default=DEFAULT_FRAGMENTS_DB): Path to fragment database
- `--dev-threshold` (float, default=2.0): Max per-signal deviation for fine matching
- `--avgdev-threshold` (float, default=1.0): Max average deviation for fine matching
- `--top` / `max_results` (int, default=5): Maximum number of fragments to return
- `--min-atoms` / `min_atom_count` (int, default=3): Minimum fragment heavy atom count
- `--verbose` (flag): Show pre-screen and fine-match counts on stderr
- `--format` / `output_format` (Choice["text", "json"], default="json"): Output format

**Implementation:**
1. Parse shifts string to `list[float]` by splitting on comma and stripping whitespace
2. Validate db_path exists (same pattern as `info` command — error + Abort if missing)
3. Open `FragmentSearcher(db_path)` as context manager
4. Call `searcher.search(shift_list, dev_threshold, avgdev_threshold, max_results, min_atom_count, verbose)`
5. Build output dict with these fields:
   - `query_shifts`: the input shift list
   - `prescreening_count`: from searcher (add a `last_prescreening_count` attribute to FragmentSearcher that is set during search — OR pass verbose=True and capture, OR add return metadata). **Simplest approach:** add `prescreening_count` and `fine_match_count` as attributes on FragmentSearcher that are set during each `search()` call.
   - `fine_match_count`: similarly from searcher
   - `result_count`: len(matches)
   - `fragments`: list of match dicts from SSCMatch.model_dump()
   - `deff_commands`: list of `"DEFF F{i} 'fragment_{i}.lsd'"` strings for i=1..len(matches)
   - `fexp_command`: `"FEXP 'F1'"` if 1 match, `"FEXP 'F1 OR F2 OR ...'"` for multiple matches, empty string if no matches

**IMPORTANT for deff_commands / fexp_command:** These are placeholder path templates. Phase 52 writes the actual fragment `.lsd` files. Phase 51 outputs syntactically correct DEFF strings with file path templates.

**Text output format:**
```
Fragment search results (N fragments from M candidates):

 Rank  Atoms  AVGDEV  SMILES
    1     13    0.17  c1ccccc1CC(C)C
    2      8    0.34  CC(C)CC(=O)O
    ...

DEFF commands:
  DEFF F1 'fragment_1.lsd'
  DEFF F2 'fragment_2.lsd'
  FEXP 'F1 OR F2'
```

**JSON output:** Pretty-printed JSON dict via `json.dumps(output, indent=2)`.

**Error handling:** If no fragments found, output `result_count: 0` with empty fragments array (not an error).

**Note on searcher metadata:** Add `prescreening_count: int = 0` and `fine_match_count: int = 0` as public attributes on `FragmentSearcher` (set in `search()` method). This avoids changing the return type. Update `searcher.py` to set these attributes during search.
