<objective>
Make 13C prediction robust to Kekulé-form SMILES (as emitted by `outlsd` / `lucy lsd run` for aromatic rings, e.g. a benzene ring written with explicit alternating double bonds), so they predict the SAME shifts as their canonical aromatic form. The incident (FIX-11) surfaced in the 2026-06-11 Opus-4.8 CASE9 run: the solution-analyst predicted on an un-aromatized Kekulé structure and nearly mis-escalated ("structure NOT in set, all implausible") — a false alarm the coordinator caught (the true structure was a genuine LSD solution; after canonicalization it ranked #1, MAE 1.17).

**Honest framing (this drives the task split):**
- The lucy-ng LIBRARY PATH WAS ALREADY CORRECT. `Chem.MolFromSmiles` with default sanitization aromatizes the real CASE9 outlsd Kekulé form, so through `predict_from_smiles` the Kekulé and canonical forms already predict identically on the real DB. The standard `predict(Kekulé) == predict(canonical)` assertion therefore PASSES even WITHOUT this plan's code change — it does NOT fail pre-fix.
- The CODE change in this plan is **hardening + pin**, not a behavioral bug fix: it makes aromatization EXPLICIT and ENFORCED (`Chem.SetAromaticity`) instead of relying on an implicit, untested RDKit default that could drift (version bump) or be broken by a future `sanitize=False` refactor. The guard test must therefore exercise the un-aromatized path on purpose (W-1) — otherwise the test is a no-op that would also pass without the fix.
- The ACTUAL fix for the CASE9 incident (which was agent-side: prediction on an un-aromatized mol) is the **compound-agnostic solution-analyst skill note** (Task 3): canonicalize/aromatize before predicting.

Purpose: prevent the false-escalation failure mode permanently (skill note) and pin the library's aromatization guarantee so it cannot silently regress (hardened, tested prepare_mol).
Output: hardened `prepare_mol`, a guard test that actually exercises restoration, a scope-documented ranker.py, and the solution-analyst skill note.
</objective>
