<objective>
Guarantee that Kekulé-form SMILES (as emitted by `outlsd` / `lucy lsd run` for aromatic rings, e.g. a benzene ring written with explicit alternating double bonds) produce the SAME 13C predictions as their canonical aromatic form. The defect (FIX-11) surfaced in the 2026-06-11 Opus-4.8 CASE9 run: the solution-analyst predicted on un-aromatized Kekulé SMILES and nearly mis-escalated ("structure NOT in set, all implausible") — a false alarm the coordinator caught (the true structure was a genuine LSD solution; after RDKit canonicalization it ranked #1, MAE 1.17).

Empirically, RDKit's `Chem.MolFromSmiles` with default sanitization ALREADY perceives aromaticity for the real CASE9 outlsd Kekulé form, so the current library path is correct-by-default. The risk is that this correctness is IMPLICIT and untested — fragile against RDKit version drift, a future `sanitize=False` refactor, or unusual ring systems RDKit's default model does not auto-aromatize. This plan makes aromatization EXPLICIT and ENFORCED at the single prediction choke point and pins it with a regression test, plus adds a compound-agnostic skill reminder.

Purpose: prevent the false-escalation failure mode permanently and self-document the aromatization guarantee.
Output: hardened `prepare_mol`, regression test, solution-analyst skill note.
</objective>
