# Phase 93 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** This phase EXTENDS the existing v9.2 webview dashboard
> (`src/lucy_ng/webview/static/index.html`) — it is not a greenfield UI. Every
> token below is extracted directly from the dashboard's current inline
> `<style>` block. Where a new value is required (tab bar, markdown-render
> typography) it is a minimal 4px-multiple / existing-hex extension, never an
> invented new palette. Tab layout itself is LOCKED by `93-CONTEXT.md` D-01
> (Run Log / 1D Spectra / 2D Spectra / Tables, right panel only; Overview +
> Structures stay a persistent non-tab left column) — this document does not
> revisit that decision, only specifies its visual/interaction treatment.

## Spacing Scale

Declared values (must be multiples of 4). Values marked "existing" are extracted verbatim from `index.html`; values marked "new" are minimal additions for this phase's tab bar, chosen to slot into the same scale.

| Token | Value | Usage | Source |
|-------|-------|-------|--------|
| xs | 4px | Tile-label internal gaps, tab-button row gap, inline code padding-y | existing (`.tile-label` gap) |
| sm | 8px | Compact spacing — status-bar padding-y, tab-button padding-y, block margins (h3, hr) | existing (`#status-bar` padding) |
| md | 16px | Default element spacing — status-bar padding-x, tab-button padding-x, h1/h2 block margins, hr margin | existing (`#status-bar` padding-x) |
| lg | 24px | Reserved for future denser sub-sections; not used by this phase's markup | new, consistent with scale |
| xl | 32px | Reserved; not used by this phase | new, consistent with scale |

Non-4px legacy values already present in the codebase and kept as-is (do not "fix" them in this phase — out of scope, low visual impact): `6px` (`.tile` padding), `10px` (`#structure-grid` gap, `#log-panel` padding-x), `12px` (`#main` gap/padding — closest 4-multiple neighbor, already effectively on-scale). These are pre-existing v9.2 values untouched by Phase 93; new elements (tab bar) must use the declared xs/sm/md tokens above, not introduce further off-scale values.

Exceptions: 44px minimum touch target — not applicable (no icon-only controls in this phase; tab buttons are full text labels with 8px/16px padding, comfortably exceeding 44px width and ~32px height including padding, acceptable for a desktop-only local dashboard).

| Role | Size | Weight | Line Height | Source |
|------|------|--------|-------------|--------|

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|

## Color

Existing dashboard has no single "brand accent" — it uses a neutral 60/30 surface split plus semantic badge colors. This phase reuses exactly those, assigning the existing primary-ink color as the 10% interactive accent (tab active-state) rather than inventing a new hex.

| Role | Value | Usage | Source |
|------|-------|-------|--------|
| Dominant (60%) | `#f8f9fa` | Page background (`body`), inline-code background, table header background | existing `body` background |
| Secondary (30%) | `#ffffff` | Card/panel surfaces — status bar, log-panel content area, structure tiles | existing `#status-bar` / `.tile` / `#log-panel` background |
| Accent (10%) | `#212529` | Active-tab underline (2px bottom border) + active-tab label text; rendered heading text (h1/h2/h3); `**bold**` label emphasis | existing primary text/ink color (`body` color, `.tile-rank`) — repurposed as the sole interactive-accent, not a new color |
| Muted text | `#6c757d` | Inactive tab label text, placeholder panel copy, hint text | existing (`#structure-hint`, `.tile-quality`, badge-waiting) |
| Hover text | `#495057` | Inactive tab hover state, `h3` agent-name headings | existing (`.label`, panel `h2`) |
| Border/divider | `#dee2e6` | Tab-bar bottom border, table cell borders, `hr`, code-block border | existing (`#status-bar` border-bottom, `.tile` border) |
| Focus ring | `#0c5460` | `:focus-visible` outline on tab buttons (keyboard accessibility) | existing (`badge-running` text color) — reused, not new |
| Destructive | not applicable | No destructive actions exist in this phase (read-only dashboard) | — |

Accent reserved for: **only** the active tab's bottom border + active tab label text, and rendered markdown heading/bold-label text. Never applied to body paragraph text, table borders, or placeholder copy.

## Component Specifications

*(Beyond the standard template columns — required to make the tab bar and markdown renderer implementable without ambiguity, per this phase's explicit deliverables.)*

### Tab bar (`#tab-bar`, replaces the static `<h2>Run Log</h2>` in `#log-panel-wrapper`)

| State | Style |
|-------|-------|
| Container | `display: flex; gap: 4px; border-bottom: 1px solid #dee2e6; flex-shrink: 0;` — sits where the old `h2` sat, same row position |
| Button (base) | `padding: 8px 16px; font-size: 13px; font-weight: 600; color: #6c757d; background: transparent; border: none; border-bottom: 2px solid transparent; cursor: pointer;` |
| Button (hover, inactive) | `color: #495057;` |
| Button (active) | `color: #212529; border-bottom-color: #212529;` |
| Button (`:focus-visible`) | `outline: 2px solid #0c5460; outline-offset: 2px;` |
| Order (fixed, locked by CONTEXT D-01) | Run Log, 1D Spectra, 2D Spectra, Tables |
| Default active on load | Run Log |
| Interaction | Click toggles `display` (`flex`/`none`) on sibling `[data-panel]` divs and `.active` class on buttons — no fetch triggered by tab switch itself; the existing 3 s `tick()` loop continues refreshing all panels' data regardless of visibility (per Research Pattern 2). No page reload, no URL/hash change (no deep-linking requirement in this phase). |

### Panel content area (each of the 4 tab panels)

Reuses the existing `#log-panel` box exactly: `background: #fff; border: 1px solid #dee2e6; border-radius: 6px; padding: 10px 12px; overflow-y: scroll; min-height: 200px; max-height: calc(100vh - 156px);` (max-height adjusted +36px from the old `calc(100vh - 120px)` to account for the new tab-bar row height — exact value non-critical, executor may tune to avoid visible clipping).

### Placeholder panels (1D Spectra / 2D Spectra / Tables — docked-in, not populated this phase)

Matches the existing `#structure-waiting` treatment: `font-size: 13px; color: #6c757d; padding: 16px 0; text-align: center;` — single centered line, no icon, no border, no card-within-card.

### Markdown-rendered Run Log panel (`renderLog()` output — see 93-RESEARCH.md Pattern 3 for the DOM-construction code)

The panel container itself is now a `<div>` (was `<pre>`); it does NOT set a page-wide `white-space: pre-wrap` or monospace font at the container level — each child node controls its own typography per the table below. Every leaf text node is set via `textContent`/`createTextNode` only (never `innerHTML`) — this is the phase's hard XSS-safety invariant (CONTEXT D-02), not a stylistic choice.

| Markdown construct | Rendered element | Style |
|---|---|---|
| `# heading` (file header, e.g. "# CASE Progress Log") | `<h1>` | `font-size: 16px; font-weight: 600; line-height: 1.2; color: #212529; margin: 16px 0 8px;` (first h1 in the document: `margin-top: 0`) |
| `## heading` (e.g. "## Setup", "## Iteration N", "## Timing Summary") | `<h2>` | `font-size: 14px; font-weight: 600; line-height: 1.2; color: #212529; margin: 16px 0 8px;` |
| `### heading` (e.g. "### NMR-Chemist", agent names) | `<h3>` | `font-size: 13px; font-weight: 600; line-height: 1.2; color: #495057; margin: 12px 0 4px;` |
| `**bold**` span (mostly `**Field:** value` labels) | `<strong>` | `font-weight: 600; color: #212529;` inline, no block change |
| `` `code` `` inline span | `<code>` | `font-family: "SFMono-Regular", Consolas, "Liberation Mono", Menlo, monospace; font-size: 12px; background: #f8f9fa; padding: 1px 4px; border-radius: 4px; color: #212529;` |
| ` ```fenced``` ` code block | `<pre><code>` | `<pre>`: `background: #f8f9fa; border: 1px solid #dee2e6; border-radius: 6px; padding: 8px 12px; overflow-x: auto; margin: 8px 0;`; `<code>`: same monospace stack, `font-size: 12px; line-height: 1.6; white-space: pre;` |
| `---` horizontal rule | `<hr>` | `border: none; border-top: 1px solid #dee2e6; margin: 16px 0;` |
| Pipe table (Timing Summary) | `<table><thead><tbody>` | `table`: `border-collapse: collapse; width: 100%; font-size: 12px; margin: 8px 0;`; `th`: `background: #f8f9fa; color: #495057; font-weight: 600; text-align: left; padding: 4px 8px; border: 1px solid #dee2e6;`; `td`: `padding: 4px 8px; border: 1px solid #dee2e6; color: #212529;` |
| Bullet list (`- item`, e.g. Team Models) | `<ul><li>` | `ul`: `margin: 4px 0 4px 20px;`; `li`: `font-size: 12px; color: #212529; line-height: 1.6;` |
| Default paragraph line (dense `**Field:** value` lines) | `<p>` | `font-size: 12px; color: #212529; line-height: 1.6; margin: 2px 0;` — tight margin matches the old `#log-panel`'s dense single-spaced feel |

Scroll-preserve behavior (D-13, existing) is unchanged in contract terms: capture `atBottom` before `renderLog()` runs, restore `scrollTop` after — see 93-RESEARCH.md Pitfall 3 and the `refreshLog()` call-site pattern. This is a behavioral/interaction contract, not a visual one, but is recorded here because it directly affects whether the panel "flickers" during the ~3 s refresh — a visual quality concern.
