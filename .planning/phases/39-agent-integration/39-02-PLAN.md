**Section structure (target ~100 lines):**

**3.6.1 Evidence Priority Table**

| Priority | Evidence Type | Trust Level | Override Policy |
|----------|---------------|-------------|-----------------|
| 1 (HIGHEST) | DEPT-135 sign | 100% | NEVER override — negative = CH2, positive = CH/CH3 |
| 2 | HSQC correlations | 95% | Only override if SNR < 10 |
| 3 | HMBC correlations | 80% | Can exclude with ELIM if conflict after thorough diagnosis |
| 4 | Chemical shift ranges | 70% | General guidelines, not rules |
| 5 (LOWEST) | Statistical detection | 60% | Override whenever NMR evidence contradicts |

**Rule:** When statistical detection contradicts higher-priority evidence, ALWAYS trust the higher-priority source. Document the conflict and resolution in CASE-PROGRESS.md.

**3.6.2 Conflict Resolution Decision Tree**

Present as a simple if/else chain the agent can follow:

```
IF detection result contradicts DEPT evidence:
    → TRUST DEPT (100%). Set MULT from DEPT, ignore detection.
    → Document: "Detection says sp2 at X ppm, DEPT shows CH2 → TRUST DEPT"

IF detection result contradicts molecular formula:
    → TRUST FORMULA (formula is given, not inferred).
    → Example: O mandatory but formula has no O → IGNORE detection.
    → Document: "Detection suggests O neighbor, formula has no O → IGNORED"

IF detection result contradicts HSQC:
    → Check spectral quality:
    → SNR > 30: TRUST HSQC
    → SNR 10-30: Use detection as tiebreaker, document uncertainty
    → SNR < 10: Consider detection, but flag as low-confidence

IF detection returns no data (no entries in database):
    → Use chemical shift heuristics as fallback:
    → 120-160 ppm: Assume sp2 (aromatic/alkene)
    → 160-220 ppm: Assume sp2 + O neighbor (carbonyl)
    → 50-90 ppm: Assume heteroatom neighbor
    → Document: "No detection data for X ppm, using shift-based heuristic"

IF detection is ambiguous (40-60% split):
    → Do NOT add constraint. Let LSD explore.
    → Document: "Detection ambiguous at X ppm (sp2=55%, sp3=45%), no constraint added"
```

**3.6.3 Worked Conflict Examples (3 minimum)**

**Example 1: Allylic CH2 — DEPT Wins Over Hybridisation Detection**
- Scenario: 13C at 132.5 ppm, DEPT-135 negative (= CH2), detection says sp2 = 92%
- Resolution: TRUST DEPT. This is CH2 adjacent to conjugated system (allylic position).
- LSD: `MULT 5 C 3 2` (sp3 CH2, NOT sp2 — DEPT is definitive)
- Progress note: "Detection suggests sp2 at 132.5 ppm, but DEPT shows CH2 → TRUST DEPT (allylic CH2)"

**Example 2: Formula Mismatch — Formula Wins Over Neighbour Detection**
- Scenario: 13C at 65.3 ppm, formula C10H16 (no oxygen), detection says O mandatory (98%)
- Resolution: IGNORE detection. Molecular formula contains no oxygen.
- LSD: No PROP constraint for O (formula forbids it)
- Progress note: "Detection suggests O at 65.3 ppm, but formula C10H16 has no O → IGNORED"

**Example 3: Peroxide Override — NMR Evidence Wins Over HHB Statistics**
- Scenario: Formula C8H14O4, shifts at 83.2/84.1 ppm (characteristic C-O-O-C), HHB says O-O forbidden (0.3%)
- Resolution: OVERRIDE statistics. Allow O-O bond. Two C-O-O shifts + 4 oxygens in formula = peroxide likely.
- LSD: Allow O-O bond (do NOT add HHB exclusion). Use `--mode relaxed` if rerunning detection.
- Progress note: "HHB says O-O forbidden, but shifts at 83-84 ppm with 4 oxygens suggest peroxide → ALLOWED"

**3.6.4 Threshold Override Guidelines**

When to use non-default thresholds:

| Situation | Action | CLI Flag |
|-----------|--------|----------|
| Default analysis | Use strict (1%/95%) | (no flag needed) |
| Rare functional group suspected | Use relaxed (0.1%/99%) | `--mode relaxed` |
| After 5+ iterations with no convergence | Loosen thresholds | `--min-frequency 0.005 --max-frequency 0.98` |
| Detection seems too restrictive | Widen shift window | `--window 3.0` (default 2.0) |
| Detection seems too vague | Try higher HOSE radius | `--radius 4` (default 3) |

**CRITICAL:** Document EVERY threshold override in CASE-PROGRESS.md with the reason. Never change thresholds silently.

**3.6.5 Statistics Augment, Never Override**

Closing statement reinforcing the hierarchy:

Statistical detection is a TOOL, not an ORACLE. It provides data-driven hints based on 7.9M database observations. These hints help reduce the LSD search space when NMR evidence is insufficient. But when NMR evidence is clear and unambiguous, statistics add nothing — and blindly following them against clear evidence creates wrong constraints.

**Think of detection as a colleague with a database**: "In my experience, carbons at 132 ppm are usually sp2." If your DEPT experiment clearly shows CH2, you politely say "thank you, but I have direct evidence" and trust the experiment.
