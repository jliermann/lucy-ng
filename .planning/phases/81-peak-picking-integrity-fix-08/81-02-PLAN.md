<objective>
Add an overcount sanity guard to both the CLI `analyze symmetry` command and the Python `SymmetryAnalysisResult` class. Currently both only handle the undercount direction (`missing_carbons > 0` → equivalence hint). When `observed_peaks > expected_carbons` (`missing_carbons < 0`), the output prints a generic "Warning: More peaks than expected carbons" without actionable language, and `SymmetryAnalysisResult.summary()` emits no alarm at all. This allowed the Phase-80 blind agent to see "confirmed" symmetry from a carbonyl-free skeleton after it manually curated 76 → 8 peaks.

Purpose: A fresh agent seeing 76 peaks vs 12 expected carbons receives a clear, unambiguous alarm: noise curation is required before any symmetry or DBE reasoning.

Output: Modified analyze.py (alarm in text + JSON) + modified symmetry_analysis.py (alarm in summary() + has_symmetry logic).
</objective>
