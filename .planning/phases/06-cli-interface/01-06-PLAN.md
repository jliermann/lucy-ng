# Phase 6: CLI Interface — Plan

## Objective

Create a command-line interface for lucy-ng that exposes all NMR processing and structure elucidation tools. The CLI enables testing, debugging, and scripting of the CASE pipeline, complementing the future MCP server for AI agent integration.

## Context

### Existing Public API

The following tools need CLI exposure:

**Readers:**
- `BrukerReader.read_1d()` — Read Bruker 1D spectra (1H, 13C, DEPT)
- `BrukerReader.read_2d()` — Read Bruker 2D spectra (HSQC, HMBC, COSY)

**Peak Picking:**
- `SimplePeakPicker.pick_peaks()` — Basic 1D peak picking
- `AdaptivePeakPicker.pick_peaks()` — Noise-adaptive 1D peak picking
- `PeakPicker2D.pick_peaks()` — Raw 2D peak picking
- `DEPTGuidedPicker.pick_hsqc_peaks()` — DEPT-guided HSQC picking (preferred)
- `HMBCGuidedPicker.pick_hmbc_peaks_from_spectra()` — Validated HMBC picking

**Analysis:**
- `SymmetryAnalyzer.analyze()` — Symmetry detection for AI reasoning
- `HydrogenBudgetAnalyzer.analyze()` — H budget comparison
- `IntensityReporter.analyze()` — HSQC intensity analysis

**Dereplication:**
- `DereplicationService.dereplicate()` — Match against nmrshiftdb

**LSD Integration:**
- `LSDInputGenerator.generate()` — Generate LSD input file
- `LSDRunner.run()` — Execute LSD solver

### CLI Framework Decision

**Use Click** (not Typer):
- Click is already widely used and well-tested
- Typer adds runtime overhead for type conversion
- Click has excellent documentation and community
- No additional dependency (already transitive through many packages)

### Command Structure

```
lucy
├── read         # Read and display spectrum info
│   ├── 1d <path>       # Read 1D spectrum
│   └── 2d <path>       # Read 2D spectrum
├── pick         # Peak picking
│   ├── 1d <path>       # Pick 1D peaks
│   ├── 2d <path>       # Pick 2D peaks
│   ├── hsqc <hsqc> <dept135> [--dept90]  # DEPT-guided HSQC
│   └── hmbc <hmbc> <c13> <hsqc>          # Guided HMBC
├── analyze      # Analysis tools
│   └── symmetry <formula> <hsqc> <dept135>  # Symmetry analysis
├── dereplicate  # Dereplication
│   └── <c13-path> <formula>   # Match against nmrshiftdb
├── lsd          # LSD integration
│   ├── generate <data-dir> <formula>  # Generate LSD input
│   └── run <input-file>               # Run LSD (if installed)
└── info         # Information
    └── version                        # Show version
```

### Output Formats

All commands support:
- `--format json` — Machine-readable JSON output
- `--format text` — Human-readable text (default)

This aligns with the AI-agent architecture: JSON for MCP integration, text for human debugging.

Create `src/lucy_ng/cli/analyze.py`:

```python
@click.group()
def analyze():
    """Analysis tools for structure elucidation."""
    pass

@analyze.command("symmetry")
@click.argument("formula")
@click.argument("hsqc_path", type=click.Path(exists=True))
@click.argument("dept135_path", type=click.Path(exists=True))
@click.option("--format", type=click.Choice(["text", "json"]), default="text")
def analyze_symmetry(formula: str, hsqc_path: str, dept135_path: str, format: str):
    """Analyze molecular symmetry from spectroscopic data."""
    hsqc = BrukerReader.read_2d(hsqc_path)
    dept135 = BrukerReader.read_1d(dept135_path)
    dept_result = DEPTGuidedPicker.pick_hsqc_peaks(hsqc, dept135)

    result = SymmetryAnalyzer.analyze(formula, dept_result, hsqc)

    if format == "json":
        click.echo(result.model_dump_json(indent=2))
    else:
        click.echo(result.summary())
```
