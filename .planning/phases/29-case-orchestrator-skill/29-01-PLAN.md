Create `~/.claude/commands/lucy-ng/case.md` — the CASE orchestrator skill that dissolves supervisor logic from `skill/supervisor/SKILL.md` into a sub-command skill.

**YAML frontmatter (follow existing sub-command pattern):**
```yaml
---
name: lucy-ng:case
description: "Orchestrate autonomous CASE agent for NMR structure elucidation. Spawns lucy-case-agent, monitors progress, detects loops, intervenes with advisory constraints."
argument-hint: "<compound_path> <formula>"
allowed-tools:
  - Bash
  - Read
  - Write
  - Glob
  - Grep
  - Task
---
```

**Skill body structure — use `<objective>`, `<process>` with named `<step>` elements, matching the pattern in status.md and dereplicate.md but extended for orchestration:**

**Step 1: parse_arguments**
- Extract compound_path and formula from arguments
- Show usage if no arguments provided:
  ```
  Usage: /lucy-ng:case <compound_path> <formula>
  Example: /lucy-ng:case data/compound/virgiline C16H21NO2
  ```

**Step 2: validate_prerequisites**
- Run `lucy --version`, `lucy lsd check`, verify database exists
- Check compound directory exists and contains NMR data
- If any check fails, report what is missing and how to fix it

**Step 3: check_dereplication_first**
- Ask user: "Would you like to run dereplication first to check for known compounds?"
- If yes: run `lucy dereplicate c13 <compound_path> <formula> --format json -n 5`
- If match found (score >= 0.95): report match and ask if user wants to proceed with full CASE anyway
- If no match or user skips: proceed to full CASE

**Step 4: spawn_case_agent**
- Use Task tool to spawn lucy-case-agent with these instructions:
  ```
  Perform CASE workflow for compound at <compound_path> with formula <formula>.

  Write CASE-PROGRESS.md in the compound directory after EVERY LSD iteration.
  Follow the incremental HMBC strategy in your inlined knowledge.
  Stop when solution_count <= 10 or after ~10 iterations.

  For detailed domain knowledge beyond your inlined content, see:
  - /Users/steinbeck/Dropbox/develop/lucy-ng/skill/SKILL.md
  - /Users/steinbeck/Dropbox/develop/lucy-ng/skill/CASE/SKILL.md
  - /Users/steinbeck/Dropbox/develop/lucy-ng/skill/diagnostic/SKILL.md
  ```
- IMPORTANT: Do NOT duplicate the agent's 613 lines of inlined knowledge. The agent already has it. Task instructions should add ONLY task-specific details (path, formula) and any advisory constraints from prior interventions.
- Wait for agent to complete or return.

**Step 5: monitor_progress**
- After agent returns, read `<compound_path>/CASE-PROGRESS.md`
- Parse all iteration entries, extracting for each:
  - Solution count
  - Constraints added (look for "ELIM" mentions)
  - Constraints removed
  - Notes (sp2 check, H budget)
  - HMBC correlations used count
- Build iteration history array for loop detection

**Step 6: detect_loops**
- Check iteration history against 4 patterns. Implement detection logic FROM skill/supervisor/SKILL.md Section 4 (dissolving it here):

  **ELIM Thrashing:**
  - Scan ALL iterations (not just recent) for "ELIM" in constraints added
  - If ELIM mentioned in 2+ iterations: detected

  **Zero-Solution Loop:**
  - Check last 3 iterations
  - If ALL have solution_count = 0: detected

  **Solution Explosion:**
  - Check last 3 iterations
  - If ALL have solution_count > 100 AND each shows < 10% reduction from previous: detected

  **Constraint Churning:**
  - Check last 5 iterations
  - Count total constraints added and removed across those 5
  - If adds > 10 AND removes > 5 AND latest solution_count > 50: detected

- If no pattern detected AND agent reported success (solutions <= 10): go to present_results
- If no pattern detected AND agent hit iteration cap: go to present_results with caveats
- If pattern detected: go to diagnose

**Step 7: diagnose**
- Before generating advisory, perform basic diagnosis (from skill/supervisor/SKILL.md Section 4 diagnostic procedures):

  **For ELIM thrashing:**
  1. Read latest LSD file, count MULT commands with hybridization = 2 (sp2)
  2. Check if sp2 count is even or odd
  3. Sum hydrogen counts from all MULT commands, compare to formula
  4. Check if recent HMBC correlations are within 1J artifact tolerance (±1.5 ppm carbon, ±0.3 ppm proton of HSQC positions)

  **For zero-solution loop:**
  1. Identify which iteration's constraints caused 0 solutions (from progress log)
  2. Check for close carbons (within 3 ppm) in the failing correlations
  3. Check for 1J artifacts in the failing batch

  **For solution explosion:**
  1. Check if ELIM command is present in LSD file
  2. Check if heteroatom constraints (BOND, LIST, PROP) are present
  3. Check quaternary carbon connectivity

  **For constraint churning:**
  1. Check if correlations are being selected by criteria or randomly
  2. Identify last known-good state (iteration with lowest non-zero solution count)

- Document diagnosis findings

**Step 8: intervene**
- Generate advisory intervention based on diagnosis. Use templates FROM skill/supervisor/SKILL.md Section 3 (Advisory Constraints):

  **ELIM thrashing advisory:**
  ```
  ELIM thrashing detected. Before retrying:
  1. Verify sp2 count is even
  2. Verify hydrogen budget matches formula
  3. Check last batch of HMBC correlations for 1J artifacts
  Do NOT add ELIM again until all checks pass.
  ```

  **Zero-solution loop advisory:**
  ```
  Zero-solution loop detected (3 consecutive iterations with 0 solutions).
  Diagnose: Remove last HMBC batch, confirm solutions return,
  test each correlation individually, check for close carbons.
  Only re-add correlations after resolving the conflict.
  ```

  **Solution explosion advisory:**
  ```
  Solution explosion stalled. Check:
  1. Remove ELIM if present
  2. Verify HMBC correlations connect NEW fragments
  3. Add heteroatom constraints (BOND or LIST/PROP)
  4. Check quaternary carbons for shift-based constraints
  ```

  **Constraint churning advisory:**
  ```
  Constraint churning detected. Reset to last known-good state.
  Follow incremental HMBC strategy: select 3-5 HIGH-CONFIDENCE
  correlations per batch. Do not add/remove randomly.
  ```

- CRITICAL: Advisory says WHAT to fix, NEVER prescribes specific LSD line edits

**Step 9: track_and_decide**
- Maintain per-pattern intervention counters (4 separate counters):
  ```
  intervention_counts = {
    ELIM_THRASHING: 0,
    ZERO_SOLUTION_LOOP: 0,
    SOLUTION_EXPLOSION: 0,
    CONSTRAINT_CHURNING: 0
  }
  ```
- Increment counter for detected pattern
- If counter reaches 10: go to escalate
- If counter < 10: go to respawn

- NOTE: This is the orchestrator's internal state tracking. Since CASE sessions are single-run, in-memory tracking is sufficient.

**Step 10: escalate**
- Present structured escalation report to user:
  ```markdown
  ## CASE Escalation Required

  **Compound:** <compound_path>
  **Formula:** <formula>
  **Pattern:** <pattern_name>
  **Intervention attempts:** <count>

  ### What Was Detected
  <Description of the loop pattern>

  ### Diagnostics Attempted
  1. <approach 1>
  2. <approach 2>
  ...

  ### Current State
  - Solution count: <count>
  - HMBC correlations used: X/Y
  - Iterations completed: N

  ### Recommendation
  <What user should investigate>
  ```
- Do NOT continue automated intervention after escalation

**Step 11: respawn**
- Re-spawn CASE agent with advisory constraints and skip-completed-work instructions:
  ```
  Resume CASE workflow for compound at <compound_path> with formula <formula>.

  ADVISORY CONSTRAINT (from orchestrator):
  <advisory text from step 8>

  Read CASE-PROGRESS.md to understand current iteration state.
  Continue from the LAST iteration number — do NOT restart from scratch.
  Apply the advisory fix, then continue normal workflow.
  ```
- After re-spawn completes: return to monitor_progress (step 5)
- This creates the supervision loop: spawn -> monitor -> detect -> diagnose -> advise -> respawn

**Step 12: present_results**
- After successful convergence or iteration cap:
  - Read CASE-PROGRESS.md for final state
  - Present ranked solutions (if agent produced them)
  - Present confidence assessment
  - List key iterations and decisions
  - Provide path to full CASE-PROGRESS.md for user review
