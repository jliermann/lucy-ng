# Phase 13-01 Plan: Database Query API

## Objective
Implement query interface for formula-based compound lookup that integrates with existing dereplication infrastructure.

## Context

The existing dereplication system uses:
- `NMRShiftDBLoader.get_by_formula()` â†’ `list[NMRShiftDBEntry]`
- `SpectrumMatcher.match_all()` expects `list[NMRShiftDBEntry]`
- `DereplicationService` coordinates loading and matching

The database module (Phase 11-12) has:
- `DatabaseManager.get_by_formula()` â†’ `list[CompoundRecord]`
- `CompoundRecord` with `ShiftRecord` list

Need: Adapter that converts `CompoundRecord` to `NMRShiftDBEntry` for compatibility.

## Tasks

### Task 1: Create Database Query Module

Create `src/lucy_ng/database/query.py`:

```python
class DatabaseQueryService:
    """Query interface compatible with existing dereplication."""

    def __init__(self, db_path: str | Path):
        """Initialize with database path."""

    def get_by_formula(self, formula: str) -> list[NMRShiftDBEntry]:
        """Query by formula, return compatible NMRShiftDBEntry objects."""

    def get_by_formulas_batch(
        self, formulas: list[str]
    ) -> dict[str, list[NMRShiftDBEntry]]:
        """Batch query for multiple formulas."""

    @staticmethod
    def compound_to_entry(compound: CompoundRecord) -> NMRShiftDBEntry:
        """Convert CompoundRecord to NMRShiftDBEntry."""
```

Key features:
- Context manager support for database connection
- Conversion of `ShiftRecord.hydrogen_count` (int) to `HydrogenCount` enum
- Efficient batch queries using single connection
- Property access pattern matching `NMRShiftDBLoader`
