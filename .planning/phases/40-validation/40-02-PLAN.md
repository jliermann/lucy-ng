  <action>
  Create `tests/test_validation_detection.py` with validation-focused tests that verify detection accuracy against known chemistry. These tests use synthetic HOSE database entries (like existing detection tests) but are specifically designed to validate the scientific correctness of detection results.

  Test cases to include:

  1. **Hybridisation accuracy for known shift regions:**
     - Aromatic region (120-160 ppm): sp2 must dominate (>90%)
     - Aliphatic region (10-50 ppm): sp3 must dominate (>90%)
     - Carbonyl region (160-220 ppm): sp2 must dominate
     - Use synthetic HOSE entries that model realistic shift distributions
     - Create a temporary SQLite database with test data for each test

  2. **Neighbour detection for known functional groups:**
     - Carbonyl C (170-220 ppm): oxygen mandatory, carbon common
     - Aromatic CH (115-145 ppm): carbon mandatory (ring neighbours)
     - Ether C (50-90 ppm): oxygen common
     - Use synthetic HOSE entries with appropriate sphere-1 elements

  3. **Signal grouping validation:**
     - Ibuprofen-like close shifts: 44.90 + 45.03 ppm should group at 0.25 ppm tolerance
     - Well-separated shifts: 44.90 + 129.38 ppm should NOT group
     - Multiplicity-aware: CH + CH2 should group (both protonated); verify multiplicity incompatibility is respected for CH vs CH3 when not ambiguous
     - Single-peak groups: isolated peaks form singleton groups

  4. **HHB detection for known formula classes:**
     - Test that hetero-hetero bond detection returns chemically specific results:
       - C-O bonds: Must be common (>50% frequency) for formulas containing oxygen (e.g., C13H18O2)
       - O-O bonds: Must be rare (<5% frequency) for non-peroxide formulas
       - N-O bonds: Present but not dominant for formulas with both N and O
       - C-N bonds: Must be common (>50% frequency) for formulas containing nitrogen
       - Pure hydrocarbons (no heteroatoms): No hetero-hetero bonds returned
     - Use synthetic bond_pair_stats entries with realistic frequency distributions

  Pattern from existing tests: Look at `tests/test_detection_hybridisation.py` and `tests/test_detection_neighbours.py` for how they set up temporary databases with synthetic HOSE entries. Follow the same pattern.

  Each test should have a docstring explaining what chemistry principle it validates.
