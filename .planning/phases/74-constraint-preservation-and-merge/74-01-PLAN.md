<objective>
Add native-equivalence emission and ring-exclusion emission to the Python API.

LSDProblem currently has no mechanism to model atom equivalence or ring exclusion.
This plan adds: (1) two new methods on LSDProblem that inject BOND/COSY into existing
lists (Option A direct injection, no new dataclass), (2) a `ring_exclusion_enabled` flag
that triggers DEFF F/FEXP emission in generate(), (3) `_write_filter_files()` in
write_file() that copies bundled ring3/ring4 alongside the .lsd file, (4) the bundled
filter files themselves as package resources, and (5) the TestNativeConstraintEmission
test class proving all of the above.

Purpose: Satisfy D-03 (native-only constraint emission) and D-04 (aromatic emergence
from constraints, not SKEL forcing). This is the Python API fix; the CASE agent's
hand-written LSD path is Phase 75.

Agent-bypass scope note: The CASE agent (lucy-lsd-engineer) writes LSD files by Bash/Write,
bypassing LSDInputGenerator. Phase 74 fixes the PROGRAMMATIC path only. RELI-02/RELI-03
are PARTIALLY satisfied here; fully proven in Phase 76 UAT after Phase 75 skill changes.

Output: modified models.py + generator.py, new filters/ package dir with ring3/ring4,
updated pyproject.toml artifacts, updated test_lsd_generator.py.
</objective>
