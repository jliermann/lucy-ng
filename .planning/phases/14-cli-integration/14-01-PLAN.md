<task type="auto">
  <name>Task 1: Update CLI to use database backend</name>
  <files>src/lucy_ng/cli/dereplicate.py</files>
  <action>
  Modify `dereplicate_c13` function to:

  1. Check for SQLite database FIRST in the path resolution:
     - `data/reference/compounds.db` (primary default)
     - Environment variable `LUCY_DATABASE` if set
     - `--database` flag if provided AND points to .db file

  2. If database found and is SQLite (.db extension):
     - Use `DatabaseQueryService` as the loader
     - Print hint to stderr about using database (e.g., "Using database: compounds.db (928K compounds)")

  3. If no database found OR `--database` points to .sd/.sdf file:
     - Fall back to existing SD file logic (COCONUT → NMRShiftDB)
     - Print hint about building database for faster queries

  4. The `DereplicationService` constructor accepts the loader - both `DatabaseQueryService` and `NMRShiftDBLoader` have `get_by_formula()` so no changes needed there.

  Key implementation notes:
  - Import `DatabaseQueryService` from `lucy_ng.database`
  - `DatabaseQueryService` is a context manager but can also be used without (call `.open()`)
  - Database path detection should happen BEFORE the existing SD file search
  - Keep existing `--database` flag behavior for explicit SD file paths
  - Add check for `.db` extension to route to database vs SD file
  </action>
  <verify>

  mv data/reference/compounds.db data/reference/compounds.db.bak
  lucy dereplicate c13 data/Ibuprofen/2 C13H18O2 2>&1 | grep -i "using"
  mv data/reference/compounds.db.bak data/reference/compounds.db
  </verify>

  - `--database` flag works for both .db and .sd files
  - Stderr message indicates which backend is being used
  </done>
