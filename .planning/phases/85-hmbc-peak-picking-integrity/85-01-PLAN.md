<objective>
Give the 2D HMBC peak picker a noise-relative SNR floor (the HMBC analog of FIX-08, which was added only to the 1D picker), expose `--snr-floor` in `lucy pick hmbc`, and route the guided-HMBC raw pick through it by default. This recovers weak-but-real long-range cross-peaks (e.g. ring-diagnostic 3J-meta aromatic correlations at ~0.6% of the global max) that the current fraction-of-max-only threshold systematically discards, while still rejecting the ~0.5-2%-of-max noise that floods a naive low-threshold pick.

Purpose: The current `lucy pick hmbc` / `PeakPicker2D.pick_peaks` threshold is fraction-of-max only (`-t`, default 0.05). Intense aliphatic/2J cross-peaks set the global max, so weak long-range correlations fall below 5%×max and are dropped — empirically proven on CASE1 (ibuprofen, HMBC expno 7): the ring-diagnostic 3J-meta cross-peaks (C≈141/H≈7.11 and C≈137/H≈7.23) sit at ~0.6% of max while the 2J-ortho are at ~22%. They only appear at `-t 0.005`, buried in a ~165-peak noise flood — no clean fraction-of-max gap. In SNR space they ARE separable. This implements FIX-12 fixes (a), (b), (c).

Output: SNR-floor 2D picker, `--snr-floor` CLI option, guided-HMBC default routing, `Peak2D.snr` field.
</objective>
