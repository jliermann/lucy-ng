In schema.py:
1. Change SCHEMA_VERSION from 4 to 5.
2. Update CREATE_HOSE_STATS_TABLE to include 5 new columns after sp1_count, all INTEGER NOT NULL DEFAULT 0:
   - has_carbon_neighbor
   - has_oxygen_neighbor
   - has_nitrogen_neighbor
   - has_sulfur_neighbor
   - has_halogen_neighbor
   These store the count of observations where the HOSE code's sphere 1 contained at least one of that element type. They are NOT boolean flags -- they are integer counts.
3. Add migration function `migrate_v4_to_v5(conn: sqlite3.Connection) -> None`:
   - Uses ALTER TABLE ADD COLUMN for each of the 5 new columns with DEFAULT 0
   - Updates schema_meta to version 5 (use str(SCHEMA_VERSION))
   - Commits the transaction
   Follow the exact pattern of migrate_v3_to_v4().

In models.py:
4. Add 5 optional fields to HOSEStatsRecord (all default to 0), after sp1_count:
   - has_carbon_neighbor: int = 0
   - has_oxygen_neighbor: int = 0
   - has_nitrogen_neighbor: int = 0
   - has_sulfur_neighbor: int = 0
   - has_halogen_neighbor: int = 0
5. Update the docstring to say "Schema v5+ includes neighbourhood detection fields."

Create NEW file src/lucy_ng/prediction/hose_parser.py:
6. Module docstring: "Parse HOSE code sphere structure for element extraction."
7. Implement `parse_sphere_1(hose_code: str) -> dict[str, int]`:
   - Splits at semicolon to get prefix and spheres part
   - Extracts sphere 1 (before first parenthesis)
   - Uses `re.findall(r'[A-Z][a-z]?', sphere1)` to extract element symbols
   - Returns dict(Counter(elements))
   - Returns empty dict for invalid/empty input
   - Import re and Counter at module level
   Exact implementation from RESEARCH.md Pattern 1.
