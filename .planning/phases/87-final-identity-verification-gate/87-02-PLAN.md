<!-- The check-identity contract delivered by 87-01 (the tool these edits consume). -->
CLI: python scripts/verify_case_solution.py check-identity --smiles '<SMILES>' [--reported-name '<name>'] [--db <path>]
Emits JSON (always exit 0):
  { mode: "identity",
    reported_name: <str|null>,
    derived: { matched: bool, name?: str, source?: "coconut"|"nmrshiftdb",
               inchi_key: str, canonical_smiles: str,
               confidence: "confirmed"|"confirmed-structure"|"novel",
               trivial_name_confirmed?: bool },
    name_match: true|false|null,
    verdict: "confirmed" | "confirmed-structure" | "tentative" | "novel",
    warning: <str|null> }
Semantics:
  - verdict "confirmed"          -> NMRShiftDB name match; analyst may state name plainly.
  - verdict "confirmed-structure"-> COCONUT accession hit; structure known, trivial name NOT confirmed -> tentative name.
  - verdict "tentative"          -> reported name disagrees with derived (or asserted name on a no-hit); warning non-null.
  - verdict "novel"              -> not in DB, no asserted name; use InChIKey + canonical SMILES as identity.
</interfaces>
