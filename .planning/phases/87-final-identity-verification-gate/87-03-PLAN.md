(it lives only in `scripts/verify_case_solution.py`, which is not installed, not on PATH, and
absent when a CASE run executes in an NMR data directory outside the repo).

duplicating it: extract the correct core into an importable package module
(`src/lucy_ng/identity.py`), add a `lucy identify` subcommand that imports it, and re-point
`scripts/verify_case_solution.py` to import the core (back-compat preserved). D-05's "one shared
deterministic path" is upheld — there is exactly one implementation, consumed by both the CLI and
the script.

Purpose: The analyst (IDENT-01) and the binding gate (IDENT-02) can finally call the tool at
runtime via `lucy identify`, which is installed and works from any data dir (cf. `lucy lsd rank`).
Output: `src/lucy_ng/identity.py`, `src/lucy_ng/cli/identify.py`, a registered `lucy identify`
command, a thinned `scripts/verify_case_solution.py`, migrated identity tests, and new CLI tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CASE-pipeline output → identity tool | The solution SMILES is semi-trusted pipeline output parsed by RDKit; invalid SMILES must degrade gracefully, never raise. |
| identity tool → reference DB | Local, read-only SQLite file (`data/reference/lucy-ng-derep.db`); the only query inputs are an InChIKey and a canonical SMILES. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-87-03-01 | Injection (Tampering) | SQL lookups in derive_identity | mitigate | Parameterized queries (`WHERE inchi_key = ?`, `WHERE smiles = ?`) carried over verbatim from 87-01; no f-string/format/`%` interpolation. Verified by grep in 87-01; preserved on move. |
| T-87-03-02 | Denial of Service | RDKit parse of untrusted SMILES | mitigate | `Chem.MolFromSmiles` returning None is handled as `{matched: False, confidence: "novel"}`; CLI always exits 0, no traceback to the user. |
| T-87-03-03 | Information Disclosure | DB path resolution / file read | accept | Read-only (`mode=ro`) open of a local bundled DB the user already owns; no remote access, no PII. Low-value local target. |
| T-87-03-SC | Tampering | npm/pip/cargo installs | n/a | No new packages installed (RDKit + sqlite3 + click already hard deps); no Package Legitimacy Gate needed. |
</threat_model>
