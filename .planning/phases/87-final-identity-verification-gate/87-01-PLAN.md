<objective>
Extend `scripts/verify_case_solution.py` with the deterministic identity-derivation core and the binding name↔structure gate. This is the single shared deterministic path (D-05) that both the analyst (derivation, IDENT-01) and the gate (verification, IDENT-02) use. It also produces the structural-identity output (InChIKey + canonical SMILES) the analyst renders for tentative names (IDENT-03).

Purpose: Eliminate the parametric naming-hallucination failure class (CASE4 chamazulene, CASE5 indigo↔isoindigo) by deriving identity from the solved structure with a tool and flagging any reported name that does not match.

Output: `derive_identity()` importable function, a `check-identity` argparse subcommand extending the existing JSON+exit-code contract, and a regression-pinned test file.
</objective>
