**Trace structure (walk through step by step):**

**Phase A: Spawn and Setup**
1. Orchestrator calls TeamCreate -> team namespace created
2. Orchestrator calls TaskCreate("peak-picking") -> task exists
3. Orchestrator calls TaskCreate("lsd-iteration-01") -> task exists
4. Orchestrator calls Task() x4 -> 4 agents spawned
5. nmr-chemist claims peak-picking task -> runs detection -> sends [SETUP-COMPLETE] to coordinator
6. Coordinator receives [SETUP-COMPLETE] -> writes ## Setup section to CASE-PROGRESS.md, retains shift list

**Phase B: Iteration 1**
7. lsd-engineer claims lsd-iteration-01 -> reads assignments from messages/CASE-PROGRESS.md
8. lsd-engineer initializes constraint inventory (Section 5C) -> writes iteration_01/compound.lsd
9. lsd-engineer sends "ready for validation" to devils-advocate
10. devils-advocate reads LSD file -> validates inventory, checks constraints -> sends [VALIDATION-PASSED] or [VALIDATION-BLOCKED]
11. Coordinator receives [VALIDATION-PASSED] -> writes DA section to CASE-PROGRESS.md
12. lsd-engineer runs LSD solver (waits for APPROVED first)
13. lsd-engineer sends [ITERATION-COMPLETE] to coordinator (solution_count = 500)
14. Coordinator receives [ITERATION-COMPLETE] -> writes LSD-Engineer section, runs detect_loops
15. detect_loops: no loop, solution_count > 10, iterations < 10 -> create lsd-iteration-02 task

Verify: Who creates lsd-iteration-02? (Must be orchestrator in monitor_progress, NOT lsd-engineer)

**Phase C: Iteration 2 (with parallel task)**
16. Orchestrator creates TaskCreate("lsd-iteration-02") + TaskCreate("hmbc-selection-02") if 10 < count <= 50
17. lsd-engineer claims lsd-iteration-02 -> reads previous LSD file -> updates inventory (Section 5D) -> writes iteration_02/compound.lsd
18. (In parallel) nmr-chemist claims hmbc-selection-02 -> selects next batch -> sends to lsd-engineer
19. lsd-engineer sends "ready for validation" -> DA validates -> [VALIDATION-PASSED]
20. lsd-engineer runs solver -> [ITERATION-COMPLETE] (solution_count = 8)
21. Coordinator: solution_count <= 10 -> creates ranking task WITH shift list from CASE-PROGRESS.md

Verify: Ranking task description contains actual shift list (not placeholder ellipsis)

**Phase D: Ranking and Completion**
22. solution-analyst claims ranking task -> reads shifts FROM TASK DESCRIPTION
23. solution-analyst runs `lucy lsd rank ... --shifts "..."` -> ranks solutions
24. solution-analyst writes analysis/final_results.md
25. solution-analyst sends [RANKING-COMPLETE] to coordinator
26. Coordinator receives [RANKING-COMPLETE] -> writes Solution-Analyst section
27. Coordinator enters present_results -> computes elapsed time -> reports results
28. Coordinator enters terminate_team -> shutdown_request x4 -> TeamDelete

Verify: Elapsed time appears in final report
