12. Final result: ranked solutions presented to user

- CASE-PROGRESS.md file exists in project root after run
- Ibuprofen structure (CC(C)Cc1ccc(cc1)C(C)C(=O)O or equivalent) appears in top 3 ranked solutions

## Test 2: Simple Sub-Commands (Secondary Validation)
### /lucy-ng:status
Invoke: /lucy-ng:status
Expected: Reports lucy-ng version, LSD availability, database status

### /lucy-ng:dereplicate
Invoke: /lucy-ng:dereplicate with Ibuprofen C13 data and formula C13H18O2
Expected: Returns top compound matches with similarity scores
Note: Ibuprofen should appear in top results since it's in the database

### /lucy-ng:predict
Invoke: /lucy-ng:predict with a known SMILES (e.g., "CC(C)Cc1ccc(cc1)C(C)C(=O)O" for Ibuprofen)
Expected: Returns predicted 13C shifts for each carbon with confidence levels

## Known Failure Modes
### Agent Not Found
Symptom: Error about agent not found or invalid agent definition
Cause: lucy-case-agent.md missing from ~/.claude/agents/ or has invalid YAML frontmatter
Fix: Verify file exists and frontmatter is valid YAML

### LSD Not Found
Symptom: Agent reports "command not found: LSD" or lucy lsd check fails
Cause: LSD solver not installed or not in PATH
Fix: Download from http://eos.univ-reims.fr/LSD/, add bin/ to PATH

### Database Missing
Symptom: Agent reports database errors during ranking
Cause: Database not downloaded or at wrong path
Fix: Run `lucy database download`

### Agent Hangs / No Progress File
Symptom: Agent runs but CASE-PROGRESS.md never appears
Cause: Agent crashed before reaching progress writing step, OR working directory mismatch prevents file creation at expected path
Check: Look for CASE-PROGRESS.md in both project root AND ~/

### Wrong Model (Sonnet instead of Opus)
Symptom: Agent responses seem shallow, makes basic NMR mistakes, doesn't show deep domain reasoning
Cause: Task tool model inheritance bug — agent got Sonnet 4 instead of Opus 4.6
Check: Claude Desktop UI may show model indicator in agent thread
Workaround: None programmatic — this is a Claude Desktop bug. Agent still works, just less capable.

### Working Directory Mismatch
Symptom: Agent reports "file not found" for paths that exist (data/Ibuprofen/)
Cause: Task-spawned agent has different cwd than orchestrator
Check: First Bash command in agent thread — if it uses relative paths and fails, this is the issue
Note: Phase 28 implemented absolute path strategy — agent SHOULD use /Users/steinbeck/Dropbox/develop/lucy-ng/

### LSD Logic Errors
Symptom: Agent gets 0 solutions or >1000 solutions and can't converge
Cause: Invalid LSD constraints (wrong atom count, bad correlations, missing ELIM)
Note: This is a skill/domain issue, not an integration issue. If orchestration detects the pattern and intervenes, the system is working correctly even if it doesn't converge on first attempt.

## Advanced Tests (Optional — Run After Ibuprofen Succeeds)
### Loop Detection
To test loop detection, you would need a dataset that naturally causes convergence problems (e.g., Virgiline/CASE7). If Ibuprofen succeeds cleanly without loops, loop detection wasn't exercised. This is expected — Ibuprofen is a "happy path" test.

### Diagnostic Delegation
Diagnostic delegation triggers after 2 failed interventions with the same pattern. This requires a genuinely difficult case that resists basic fixes. Consider testing with harder datasets if available.

- Whether agent thread was visible in Claude Desktop
- Approximate runtime before failure
- Any other observations about agent behavior
