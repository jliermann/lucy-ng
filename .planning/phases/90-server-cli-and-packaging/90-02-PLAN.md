<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI caller → server.start(analysis_dir) | `analysis_dir` is user-supplied; crosses into filesystem + subprocess args |
| server.py → OS process (uvicorn subprocess) | Detached child process tracked only by PID in `.webview.json` |
| loopback network → FastAPI app | Bound to 127.0.0.1 only; no auth (localhost trust) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-90-03 | Tampering | Path traversal via `analysis_dir` in `start`/`status`/`stop` | mitigate | `Path.resolve()` + verify `is_dir()` before launch; state file confined to the resolved dir |
| T-90-04 | Tampering | Subprocess arg injection via `analysis_dir` | mitigate | `subprocess.Popen` LIST form only, never `shell=True`; `str(Path.resolve())` passed as a single argv element |
| T-90-05 | Denial of Service | Port squatting (another proc binds chosen port before uvicorn) | mitigate | Poll `proc.poll()` ~0.5s after launch; raise RuntimeError on early exit instead of writing a dead-port state file |
| T-90-06 | Denial of Service | Stale `.webview.json` after SIGKILL/crash causes false "running" | mitigate | `status()` probes `is_alive()` and removes stale file; `stop()` escalates SIGTERM→SIGKILL |
| T-90-07 | Information Disclosure | FastAPI auto-docs (`/docs`, `/redoc`) expose schema | accept/mitigate | `docs_url=None, redoc_url=None`; localhost-only bind means low value even if enabled |
| T-90-SC | Tampering | npm/pip/cargo installs | mitigate | No installs in this plan. fastapi/uvicorn audited [OK] in 90-RESEARCH Package Legitimacy Audit; no [ASSUMED]/[SUS] |
</threat_model>
