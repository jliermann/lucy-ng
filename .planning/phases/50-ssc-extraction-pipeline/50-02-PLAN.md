Create `src/lucy_ng/fragments/extractor.py` with two components:

**Component A: Pure fragmentation algorithm (module-level functions)**

1. `extract_fragments_for_compound(smiles: str, atom_shifts: list[tuple[int, float]]) -> list[SSCRecord]`
   - Parse SMILES with `Chem.MolFromSmiles(smiles)` — NO `AddHs()` (implicit H only, per CLAUDE.md critical architecture decision)
   - Standardize aromaticity: `Chem.SetAromaticity(mol, Chem.AromaticityModel.AROMATICITY_MDL)` — MANDATORY before any fragmentation
   - Build `shift_map: dict[int, float]` from atom_shifts (skip entries where atom_index is None — research Pitfall 6)
   - For each atom with a known shift, extract atom-centered environments at radius 1, 2, 3 (max radius 3 per Wenk thesis)
   - For each ring system (up to size 6), extract ring-centered environment at radius 1 (ring atoms + immediate neighbors)
   - Deduplicate by canonical SMILES within the compound (in-memory set — this is per-compound, NOT global)
   - For each unique fragment: collect shifts of fragment atoms from shift_map, compute avg/min/max, generate fingerprint bitset via `shifts_to_fingerprint`, return SSCRecord

2. `_extract_atom_environment(mol, atom_idx, radius, shift_map) -> tuple[str | None, list[float]]`
   - Use `Chem.FindAtomEnvironmentOfRadiusN(mol, radius, atom_idx)` to get bond environment
   - Use `Chem.PathToSubmol(mol, env, atomMap=amap)` to extract fragment molecule
   - Collect shifts for all original atoms in the fragment that have known shifts (via amap keys)
   - Return `(canonical_smiles, shifts)` or `(None, [])` if extraction fails

3. `_extract_ring_environment(mol, ring_atoms, radius, shift_map) -> tuple[str | None, list[float]]`
   - Get all bonds between ring atoms: `mol.GetBondBetweenAtoms(a, b)` for all pairs in ring
   - Expand by radius 1: for each ring atom, include bonds to immediate non-ring neighbors (use `atom.GetBonds()`)
   - Build submol from collected bond indices, collect shifts from shift_map
   - Return `(canonical_smiles, shifts)` or `(None, [])` if no shifts found

4. `_build_ssc_record(smiles: str, shifts: list[float]) -> SSCRecord`
   - Helper: creates SSCRecord with computed atom_count (from `Chem.MolFromSmiles(smiles).GetNumAtoms()`), avg/min/max shift, and fingerprint bitset from `shifts_to_fingerprint(shifts)`

The `run()` method follows the `ResumableHOSEStatsGenerator` pattern from `stats_generator.py`:

1. If `fresh`: call `fragment_db.clear_ssc_data()` (truncates tables + clears checkpoints), start from `compound_id=0`
2. If `resume` (default): load `CK_LAST_ID` from checkpoint, start from that ID. Also restore counters from checkpoint for accurate progress reporting.
3. Iterate `compound_db.iter_compounds_with_shifts_from(start_id=start_id)`:
   - Filter: skip compounds where ALL shifts have `atom_index=None` — log to stderr: `SKIPPED: compound_id={id} (no atom-indexed shifts)` (required by success criteria)
   - Call `extract_fragments_for_compound(smiles, indexed_shifts)` for each compound
   - Batch collect fragments, insert with `fragment_db.insert_ssc_batch(batch)` every `chunk_size` compounds
   - CRITICAL ORDER: insert SSC batch FIRST, then save checkpoint (research Pitfall 4)
   - If `sample` is set, stop after processing that many compounds
4. Progress bar: use `tqdm` with `desc="Extracting SSCs"`, update after each compound
5. Final checkpoint save after loop exits
6. Return `SSCExtractionResult` with all counters

          sample: int | None, resume: bool) -> None:
    """Build fragment (SSC) database from compound database.

    Extracts substructure-subspectrum correlations from all compounds with

    COMPOUND_DB: Path to lucy-ng-derep.db (source compounds).
    FRAGMENT_DB: Path to lucy-ng-fragments.db (output, created if not exists).

    Examples:
        # Validate bin size on 1000 compounds
        lucy fragment build data/reference/lucy-ng-derep.db --sample 1000

        # Full extraction (resumable)
        lucy fragment build data/reference/lucy-ng-derep.db

        # Restart from scratch
        lucy fragment build data/reference/lucy-ng-derep.db --fresh
    """

2. In the `build` function:
   - Open both databases as context managers: `DatabaseManager(compound_db)` and `FragmentDatabaseManager(fragment_db)`
   - Call `fragment_db.create_tables()` to ensure schema exists
   - Set `fresh = not resume` (Click's `--resume/--fresh` flag gives `resume=True` for `--resume`, `resume=False` for `--fresh`)
   - Create `SSCExtractor(compound_db=compound_db_mgr, fragment_db=fragment_db_mgr)`
   - Call `result = extractor.run(chunk_size=chunk_size, sample=sample, resume=resume, fresh=not resume if not resume else False)` — Note: fresh is the inverse of resume when `--fresh` is passed
   - Actually, simplify: the Click `--resume/--fresh` flag means `resume=True` by default and `resume=False` when `--fresh` is passed. Map to extractor: `extractor.run(chunk_size=chunk_size, sample=sample, resume=resume, fresh=not resume)`
   - Print result summary: compounds processed, skipped, SSCs extracted, SSCs duplicate
   - If `sample` was specified AND `sample >= 100`: run self-search recall validation
3. Self-search recall validation (when `--sample` is used and sample >= 100):

   - Report recall percentage: `click.echo(f"Self-search recall: {recall:.1%} ({hits}/{total})")`
