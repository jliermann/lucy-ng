Run the v4.0 CASE team by invoking the orchestrator skill:

```
/lucy-ng:case data/Ibuprofen C13H18O2
```

This will:
- Spawn 4 specialist agents (nmr-chemist, lsd-engineer, solution-analyst, devils-advocate)
- NMR-Chemist picks peaks, runs statistical detection
- LSD-Engineer builds LSD files with constraint inventory
- Devils-Advocate validates each LSD file before solver runs
- Solution-Analyst ranks solutions after convergence
- Orchestrator monitors progress, detects loops, intervenes if needed

**IMPORTANT: Let the team run autonomously.** Do NOT intervene unless something catastrophically fails (e.g., all agents crash, no messages for 10+ minutes). The purpose of this UAT is to observe the team's autonomous behavior, including whether it self-corrects through the Devils-Advocate gate.

**Expected behavior:**
- DA blocks are EXPECTED and indicate the system is working (Bug 1-5 catches)
- Up to 8 iterations is acceptable (v3.0 was 4, v4.0 adds DA gate overhead)
- Orchestrator advisory interventions indicate loop detection is working
- Team should converge to a manageable solution count and present final ranking

**Completion signals:**
- Orchestrator presents final results
- analysis/final_results.md exists with ranked solutions
- CASE-PROGRESS.md shows complete iteration history

If the run times out or the team gets stuck in a loop for >10 iterations, the orchestrator's built-in loop detection and escalation should handle it. If it does not, document the failure mode.
