CHECKPOINT: The user must run CASE elucidation on each compound. Each compound needs TWO runs (baseline and fragment-enhanced). CASE runs are interactive and time-consuming (each spawns the 4-agent team via /lucy-ng:case).

**Procedure for each compound:**

**Run 1 -- Baseline (no fragments):**
1. Temporarily move the fragment DB: `mv data/reference/lucy-ng-fragments.db data/reference/lucy-ng-fragments.db.bak`
2. Run CASE: `/lucy-ng:case data/{compound} {formula}`
3. Record solution count from final iteration in CASE-PROGRESS.md
4. Record whether aromatic ring warning appears in ranking output
5. Restore fragment DB: `mv data/reference/lucy-ng-fragments.db.bak data/reference/lucy-ng-fragments.db`

**Run 2 -- With fragments:**
1. Ensure fragment DB is in place: `ls data/reference/lucy-ng-fragments.db`
2. Run CASE: `/lucy-ng:case data/{compound} {formula}`
3. Record solution count, fragment applied (SMILES), fragment conflicts
4. Record whether correct structure appears in ranked solutions

**Compound commands (copy-paste ready):**
```
/lucy-ng:case data/Ibuprofen C13H18O2
/lucy-ng:case data/MC047_9 C10H14O2
/lucy-ng:case data/PSP C12H12O5
/lucy-ng:case data/Sali_Eth C18H20N2
/lucy-ng:case "data/4-Hydroxy-3-Iodo-biphenyl" C12H9IO
/lucy-ng:case "data/4-(1-Hydroxyethyl)benzoic acid isopropylester" C12H16O3
```

**Diagnosis criteria per compound:**
- If solutions lack aromatic rings AND 4+ carbons are in 110-160 ppm range: diagnose as "4J HMBC failure"
- If solutions have correct ring types but wrong connectivity: diagnose as "fragment library gap"
- If correct structure found in top N: diagnose as "success"
- If fragment was discarded (zero solutions): log as "fragment conflict"
