<objective>
Implement `detect_aromatic_cosy_pairs()` in generator.py and the `lucy detect aromatic-cosy` CLI subcommand (FIX-02). This fixes the Phase 76 failure where the agent hand-derived COSY pairs incorrectly (emitted adjacent within-group COSY 4 5 / COSY 6 7 instead of cross-ring COSY 4 7 / COSY 5 6), preventing aromatic ring emergence. The deterministic algorithm uses `zip(sorted(groupA.atom_ids), reversed(sorted(groupB.atom_ids)))`, verified by Arm A reference.

This plan depends on Plan 01 (77-01-PLAN.md) for its end-to-end integration test: the arm_a_ring_excl.lsd fixture requires the filter-file-copy fix to be in place before the full pipeline can produce aromatic solutions.

Purpose: Remove the entire error class of manual atom-index reasoning from the agent. The agent calls `lucy detect aromatic-cosy` and gets ready-to-use `COSY a b` lines; it no longer needs to understand the reversed-pairing rule or para-ring topology.

Output: detect_aromatic_cosy_pairs() function + lucy detect aromatic-cosy CLI + unit tests + integration test confirming aromatic ring emerges end-to-end.
</objective>
