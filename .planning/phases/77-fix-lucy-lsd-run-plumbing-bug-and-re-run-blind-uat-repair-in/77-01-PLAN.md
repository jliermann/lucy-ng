<objective>
Fix the `lucy lsd run` false-success bug (FIX-01): repair `_execute_lsd` so it copies bundled ring3/ring4 filter files alongside the LSD input file, and harden `_invoke_outlsd` to raise `RuntimeError` on known-bad outlsd output instead of writing garbage to `solutions.smi` and returning a non-None path. Add regression tests covering both paths.

Purpose: The Phase 76 UAT agent received `outlsd: This is not a file for OUTLSD.` in solutions.smi while the runner reported success. The root cause is that `_execute_lsd` copies the `.lsd` file to output_dir but does not copy the ring3/ring4 filter files that every LSD file emitted by `LSDInputGenerator` references via `DEFF F1 "ring3"` / `DEFF F2 "ring4"`. LSD writes a partial `.sol` (input echo only) when these files are missing. `_invoke_outlsd` then sees non-empty stdout and writes the error string to solutions.smi as "success". This plan is the prerequisite for Plan 02's end-to-end integration test.

Output: Fixed runner.py + new `TestInvokeOutlsd` unit tests + `test_ring_exclusion_lsd_produces_smiles` integration test + `arm_a_ring_excl.lsd` fixture.
</objective>
