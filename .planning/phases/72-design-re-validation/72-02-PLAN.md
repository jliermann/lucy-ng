    ## Q3 Answer: Constraint Translation (D-03)

    **Verdict:** The LSD-file generator emits native-only commands. Translation happens at GENERATION
    time. No emitted LSD file may contain SYME or DEFF NOT.

    **SYME translation correction — CRITICAL:**
    CONTEXT.md D-03 states "SYME → DUPL" — this is INCORRECT. DUPL is output-deduplication
    (DUPL 2 = default behavior, removes isomorphic structures from LSD output). DUPL does NOT
    mark atoms as structurally equivalent for HMBC interpretation; it cannot replace SYME's intended
    role. The correct native mechanism for atom equivalence in the iter3 working file is structural:
    - Gem-dimethyl equivalence: BOND 10 11 + BOND 10 12 (forces both CH3 groups to attach to the same CH)
    - Aromatic CH pair equivalence: COSY 4 7 + COSY 5 6 (encodes the H-H coupling that implies ring adjacency)
    These BOND/COSY constraints enforce the same topology that SYME was intended to encode.

    **Native DEFF NOT translation (confirmed working in iter3):**
    - DEFF NOT C1CC1 / DEFF NOT C1CCC1 (and heteroatom variants) → DEFF F1 ring3 + DEFF F2 ring4 + FEXP "NOT F1 AND NOT F2"
    - Filter files at /Users/steinbeck/Dropbox/develop/LSD/Filters/ring3 and ring4 (pre-built, verified)
    - DEFF paths must be absolute OR relative to the LSD invocation CWD

    **Open sub-question for Phase 74:** Is BOND+COSY always the right SYME translation, or are there
    SYME use cases (e.g., homotopic CH2 protons) where BOND/COSY are not appropriate? Phase 74 must
    decide and implement. The decision document flags this as an implementation detail for Phase 74,
    not a blocker for DESIGN-01/DESIGN-02 sign-off.

    → Phase 74: Rewrite generator.py to emit native-only; implement BOND/COSY for SYME; DEFF F/FEXP for DEFF NOT
