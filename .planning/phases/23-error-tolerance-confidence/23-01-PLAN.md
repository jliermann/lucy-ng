**11.1 Close Carbon Detection (ETOL-01, ETOL-03)**

Resolution-based ambiguity detection. Key content:

- Calculate digital resolution independently for 1D 13C, HSQC F1 (carbon axis), and HMBC F1 (carbon axis)
- Formula: `resolution = len(ppm_scale) / (ppm_max - ppm_min)` = pts/ppm
- Minimum distinguishable spacing: `min_spacing = 1.5 / resolution_pts_per_ppm`
- Two carbons are unresolvable if `abs(shift_A - shift_B) < min_spacing` in ANY dimension
- Check all dimensions independently -- each has different resolution
- Reference Section 2 quality tiers: >10 pts/ppm = 0.15 ppm spacing, 5-10 = 0.30, 2-5 = 0.75, <2 = 1.5+
- Note: this replaces hard-coded ppm thresholds (rejected by user as not physically grounded)
- Include statement that this detection is designed for future augmentation by an atom environment database

When carbons are unresolvable:
- Use LSD LIST/PROP mechanism to encode ambiguity in a SINGLE LSD file (NOT separate variant files)
- Show concrete LSD example:
  ```
  MULT 5 C 2 0   ; could be 155.08 or 155.32 ppm
  MULT 6 C 2 0   ; could be 155.08 or 155.32 ppm
  LIST L1 5 6    ; group unresolvable carbons
  ; When HMBC correlates to one of them:
  ; Use PROP to express "at least one atom in L1 connects to proton X"
  ```
- Document in Ambiguities Detected section (see 11.4 below)

**11.2 DEPT/HSQC Multiplicity Conflict Resolution (ETOL-02)**

Context-dependent conflict resolution. Key content:

- No blanket rule -- NOT "DEPT always wins" or "HSQC always wins"
- Priority-ordered decision tree:
  1. DEPT-90 availability (highest priority): DEPT-90 shows ONLY CH -- near-definitive. If available, trust over HSQC pattern inference for CH identification
  2. S/N comparison: Higher S/N experiment is more trustworthy. DEPT SNR > 50 with HSQC SNR < 20 -> trust DEPT. Reverse -> trust HSQC
  3. Chemical shift expectations: < 30 ppm -> likely CH3, 100-160 ppm -> likely aromatic CH, 50-90 ppm negative DEPT -> likely O-CH2
  4. Consistency check: HMBC correlation count (CH typical 2-4, CH2 2-3, CH3 1-2), hydrogen budget from formula
- Edge case: BOTH experiments have poor S/N (< 20): mark as explicitly ambiguous, assign Low confidence, use shift-based heuristic as last resort, suggest re-acquisition
- Resolution: Resolve to ONE multiplicity, document the alternative in Ambiguities section (do NOT branch into separate LSD runs)
- Document ALL disagreements, even minor ones (audit trail)

**11.3 Quaternary Carbon HMBC Sparsity (ETOL-04)**

Handling quaternary carbons with 0-1 HMBC correlations. Key content:

Shift-based constraint mapping table (explicitly modular for future atom environment database replacement):

| Shift Range (ppm) | Likely Environment | LSD Constraint |
|-------------------|-------------------|----------------|
| 160-180 | Carboxylic acid/ester/amide C=O | BOND to oxygen |
| 180-220 | Ketone/aldehyde C=O | BOND to oxygen |
| 120-160 (aromatic context) | Aromatic junction | Constrain to aromatic ring carbons |
| < 50 | Quaternary aliphatic | Rare (tert-butyl); note as unusual |

Note: "This mapping is heuristic and designed for replacement by a future atom environment database. The specific shift ranges and constraint types should be treated as initial guidelines."

Single HMBC correlation: treat with HIGHER confidence (precious, not suspicious -- it is the only connectivity info for this atom)

Targeted HMBC threshold reduction for 0-1 correlation quaternary carbons:
- Start at current threshold from guided picking
- Reduce by 20% per step (NOT 50% -- user explicitly preferred gradual)
- Re-examine HMBC in ±2.5 ppm window around quaternary carbon shift
- Validate new peaks against 13C and HSQC (guided picking logic)
- Stop when: correlations found, OR 3 consecutive reductions yield 0 new validated peaks (diminishing returns), OR threshold reaches `noise_floor x 2` (reasonable floor)
- The floor is determined by Claude based on noise characteristics of the specific spectrum

**11.4 Ambiguities Detected Output Section**

Standardized output format. Key content:

All detected ambiguities must be documented in a dedicated "Ambiguities Detected" section in the analysis output. Use table format:

```markdown
## Ambiguities Detected

| Carbon/Issue | Type | Resolution Detail | Impact on Constraints |
|-------------|------|-------------------|----------------------|
| 155.08 / 155.32 ppm | Close carbons | HMBC F1: 4.2 pts/ppm, 0.36 ppm limit, 0.24 ppm apart | Used LIST/PROP for C7-H12 correlation |
| 28.5 ppm | DEPT/HSQC conflict | DEPT-135 positive, HSQC CH3 pattern, no DEPT-90 | Assigned CH3 (shift < 30 ppm), alternative CH possible |
| 172.4 ppm (C=O) | Sparse HMBC | 0 correlations after threshold reduction to 0.025 | BOND to oxygen (172 ppm carbonyl region) |
```

Each entry must include:
- The specific carbon or carbon pair
- Type of ambiguity (Close carbons / DEPT-HSQC conflict / Sparse HMBC / Other)
- Quantitative resolution detail (pts/ppm, spacing limit, S/N ratios, threshold values)
- Specific impact on LSD constraints (what was done and what the alternative would be)

This section is MANDATORY whenever any ambiguity is detected. If zero ambiguities: state "No ambiguities detected" explicitly.
